### 7a. Actionable

**單樣本期 (5/1-6/12) + 9 個月擴大樣本，都沒有任何 setup 達 actionable 門檻。**

結論：既有 xiaoge detector + 條件 stack 變體都無法在 C6 Rule A 出場規則下達到「勝率 ≥ 65%」。

根本原因分析：

1. **C6 Rule A 跟 xiaoge 偵測物理意義不匹配**：
   - C6 Rule A = 收 < MA10 by 2% 就出、平均持 7-9 天
   - xiaoge detector 抓「籌碼集中 + 月線多頭」、物理意義是「至少抱數週」
   - Rule A 在強股第一次回測 MA10 就 cut、抓不到第二波
   - 證據：擴大窗口 ~9 個月、所有 detector wr 都壓在 32-36%、跟主窗口一致
2. **C6 vs upper_band 對比**：
   - upper_band 出場：D2v1_chip_10pct wr=46.9%、avg=+1.65%、avg_hold 6.4d
   - C6 出場：同 setup wr=35.7%、avg=+2.71%、avg_hold 8.6d
   - C6 把「短期回擋但後續續強」的 trade 全切掉、avg_ret 變高但 wr 大幅下降
   - 表示 detector 本身能抓到大 winner（C6 max_ret 167% vs upper 125%）、
     但中段震盪期被 cut
3. **Rule A 對 xiaoge 是錯的 exit 選擇**：xiaoge 出場應該回到 `leave_upper_band`、
   或設計新的 xiaoge-style exit（close < bb_mid 或 close < ma20）

**最終建議：**
- **不要用 C6 出場 + xiaoge detector 直接上 production**
- 維持 xiaoge 既有 `leave_upper_band` 出場機制
- 若要用 C6 規則（zhuli 派系），detector 必須改成 trend follow + breakout 型
  （見「6a. 盲點補救建議」momentum_red3d 系列）
