## 3. 衝突點與隔離原則

### 3.1 命名衝突

| 衝突項目 | 說明 | 解法 |
|---|---|---|
| 「形態一」 | K線力量課程有「型態一（假跌破）」；主力大課程有「形態一（奇形/均線多頭）」，兩者截然不同 | 所有主力大檔案、函式、變數名稱一律加 `zhuli_` 前綴 |
| 「type1 / pattern1」 | Python 檔內部常數若用通用名稱會混淆 | 主力大常數用 `ZHULI_FORM1_*` 前綴 |
| `index.json` 的 `category` 欄位 | K線力量用「入門/行進ing/型態學」作為子分類 | 主力大用「scripts（講稿）」/ `pressplay_weekly`（本週市場資訊報）/ `pressplay_video`（純影片）三類 |

### 3.2 概念衝突

| 衝突項目 | 主力大定義 | 注意事項 |
|---|---|---|
| 停損邏輯 | 以均線（5ma/20ma）、缺口、量價背離為主要訊號；收盤確認 | 兩套課程的停損訊號名稱可能相似，但觸發條件和臨界值不同，絕不能互相引用 |
| 「攻擊」概念 | 主力大無明確「攻擊形態」定義，用「拉抬/整理/出貨」框架描述主力意圖 | 不可把 K線力量的「攻擊 K」定義代入主力大的拉抬判讀 |
| 進場時機 | 主力大依不同策略有三種進場時機（保守/進取/激進），飆股奇形詳述 | 時機定義各套課程獨立，不得跨課程混用 |

## 7. 不建議的整合動作

以下為絕對禁止項目：

- **禁止** 從 `docs/K線力量判斷入門/` 引用任何教學內容放入主力大文件
- **禁止** 把 K線力量的「型態一（假跌破）」定義與主力大的「形態一（奇形）」或「形態一（均線多頭排列）」放在同一個文件或同一個函式裡
- **禁止** 把兩套課程的相似概念（如「均線支撐」「量縮回跌」）在沒有明確標示來源的情況下合寫成通用規則
- **禁止** 因為講稿缺漏而自行補完形態二/三/四的定義（哪怕聽起來合理）；只能標注「待補」
- **禁止** 把窒息量的量化閾值自行設定（如「近 5 日均量的 30% 以下」），除非老師明確說過此數字
- **禁止** 把 PressPlay 市場資訊報的「老師當週操作水位（如 5.5~7 成）」寫成固定策略規則；這是主觀動態判斷，不是回測條件
- **禁止** 在 scripts/ 的主力大 scanner 內直接 import K線力量 scanner 的任何函式（底層基礎設施如 `kline_course_backtest.py` 可共用，但不能 import 另一套課程的 strategy_check）
- **禁止** 把「處置股三種固定發展買賣點」（月直播課，定義在 YouTube 影片）寫成文字定義，除非有完整影片逐字稿
- **不建議** 在 `index.json` 的 `category` 欄位使用與 K線力量相同的子分類名稱（如「入門」「型態學」），避免混淆
