4. **非 intraday scanner 進場 6282/6285 = 高風險** — 所有非 intraday 的 stop_loss 出場平均 −7.3%（6282）/ −7.6%（6285），而 intraday 同標的平均 +0.65% / +0.25%。

**修復方向：**
- reversal_breakout / pennant_flag 對網通族群（高波動、趨勢性強）的停損設置太寬（允許 10+ 天持倉）；或訊號本身在這類強趨勢股中容易「假訊號後大跌」。
- 可考慮對這兩檔在 reversal_breakout/pennant_flag scanner 加設 hold_days ≤ 3 的截止，或直接排除這兩個 scanner 對這類高波動網通/衛星股的訊號。

## 3. 可行診斷與修復方向

### 診斷 A：6282/6285 exit rule 問題

**根本原因**：不是出場太早，而是 reversal_breakout / pennant_flag 這兩個 scanner 對高 beta 趨勢股（網通/衛星）的停損幅度設計假設了「回測型」股性，但這類股票急跌時單日 −8~10% 不罕見。

**修復方向（不涉及課程外條件）**：
- 在 reversal_breakout / pennant_flag scanner 的持倉上限從目前的寬鬆設定降至 ≤ 3 天（overnight_exit 的 intraday 訊號在這兩檔反而表現正報酬）。
- 或：對 6282/6285 所屬族群（網通/低軌衛星）在這兩個 scanner 的訊號加重樣本審查。

### 診斷 B：被動元件未捕捉問題

**根本原因**：被動元件在 2026 年的行情是「年初洗盤 → 長期趨勢大漲」型態。Scanner 以短線進出為設計核心：
1. 年初洗盤期觸發進場訊號 → 被停損洗出
2. 停損後無重新追蹤機制 → 錯失後續整段行情

**修復方向**：
- 被動族群需要的不是更多短線訊號，而是識別「年初盤整後站穩 MA20 + 籌碼確認」的中期進場訊號。
- 現有 swing_breakout / suffocation 設計假設股票已在高速行進中，對「剛從底部啟動」的型態識別不足。
- 如需捕捉，可考慮加入「低點墊高 + MA20 從下彎轉平」型別的 detector，但這屬於課程框架擴充，需先確認課程依據。
