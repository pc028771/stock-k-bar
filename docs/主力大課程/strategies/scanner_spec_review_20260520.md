## Executive Summary

1. **scanner 9 條件覆蓋了「整理 + 多頭排列」骨幹、但完全沒實作老師 5/20 整堂課的核心教學**：N 字段位區分（標準型 vs 快速型）、扣抵值預判 5MA 方向、小結構整理的「平台守區間」識別。
2. **扣抵值是老師 5/20 最重的單一教學點**（22 次提及 + 「值得這個課了」結論句）、scanner 目前 0 實作。建議新增 `kouvalue_next_day_5ma_up_ok` 條件 — 純用昨日/今日資料即可預判明日 5MA 方向、可作為「pre-trigger」加分項。
3. **「N 字第二段才咬進去」是 timing 鐵則**、scanner 目前無區段判斷、可能在「N 字第一段拉完」就誤觸發。這解釋了為何 5/20 群創進到 strict 命中清單（背景脈絡 = N 字第一段已結束、進入小結構整理 = 對的 timing）、但 scanner 並非「故意」抓到、而是 incidental。
4. **「小結構整理」= 平台守區間 ≥ 4-8 日**、scanner 目前 `consolidation_days=4` 只是註解 (備注用、未實作於 detect)。需把整理日數變成 hard filter（最近 N 日 high-low range 收斂 + 收盤守在 MA5/MA10 區間）。
5. **「不要用大可不必守在這個地方」位階判斷**（量體大 + 拉兩根半要守不容易）= 老師對「股本大小 vs 量能」的隱含條件、scanner 沒實作、暫不建議實作（老師原話模糊、需後續課程確認）。

## 3. N 字 vs 小結構 — 是否該區分

### 老師明示是「不同型態」

> 「他可以用很多種方式、一個是小結構整理、一個是快速 N 字就結束」[00:51–00:53]
> 「這次不一樣跟上次……我這個時候就可以判斷他不是要做 N 字標準形態」[03:11–03:17]

### 兩個型態的 SOP 差異

| 維度 | 小結構整理（標準）| 快速 N 字 |
|---|---|---|
| 整理時間 | 7–8 日（老師原話 [02:08–02:13]） | 1–3 日（「123 根下來熟一下」[01:04]）|
| 進場時機 | 整理結束、突破平台 | N 字第二段起漲 |
| 倉位 | 可壓較多（「整理夠久 = 籌碼乾淨」） | 老師原話：**「我沒辦法壓太多的部位進去」** [01:28]（量體大 + 短線）|
| 留倉時間 | 波段 | 「很短線的情況」[01:13–01:16] |

### 建議

- **scanner 應該分兩個模式輸出**：`pattern: "small_structure"` vs `pattern: "fast_n"` —— 不是把 9 條件改、而是在 hit dict 多一個分類欄位、後續對應不同 sizing/holding。
- **判斷規則（依老師原話）**：
  - `small_structure`: 最近 6–10 個交易日 close 在某個 ±3% 區間內波動 + 守 5MA + 5MA 上揚或持平
  - `fast_n`: 最近 1–3 個交易日從近期 high 回測 5MA、然後當日反彈 close 在 5MA 之上
- **實作建議優先 small_structure**（老師說 fast N 字「不太壓部位」、scanner 主要是給 swing entry 用）

## 4. 扣抵值是否該加進 Scanner

### 老師原話核心（22 次提及、整堂課最重）

> 「扣抵值是唯一可以預測的東西」[08:44]
> 「明天只要一開盤、扣抵值就會扣到這一根、就會扣到這一根」[08:31–08:34]
> 「扣抵值的比較是跟扣抵值的『前一根』比較、不是扣抵值本身」[05:52–06:00]
> 「明天只要開盤不要低於 35.4、它的五日線保證往上彎」[08:54–09:00]
> 「上揚的均線跟下揚的均線哪一個壓力比較小？上揚的」[10:05–10:11]

### 扣抵值公式（老師明示）

明日 5MA 方向預判：
- 明日 5MA 上彎 ⇔ 明日收盤 > **6 日前** close（即明日扣抵 = 5 日前 close、比較對象是 5 日前 close 的「前一根」= 6 日前 close）
- 老師例子：群創今日 = 36.95、明日扣抵會「移到 35.4」 → 明日只要 open > 35.4 (代表 close ≥ 5 日前 close)、5MA 即上揚
- **注意**：老師原話「不要低於 35.4」用 open 比較、嚴格定義應該是 close 比較、但老師用 open 作為盤中即時預判依據

### Implementation 建議

新增 DEFAULT_CFG 欄位：

```
"require_5ma_kou_up_next_day": False,  # 是否強制要求明日扣抵值預判 5MA 會上揚
"kou_compare_window": 5,                # 5MA 對應 5 日窗口
```

新增 detect 內欄位：
- `kou_5d_prev_close`: 6 日前 close（明日扣抵值 — 比較基準）
- `next_day_5ma_up_threshold`: 6 日前 close 數值（明日 open 高於此值、5MA 必上揚）
- `next_day_5ma_up_ok`: 今日 close > 6 日前 close → True（不需等明日、用今日 close 也可預判方向變動概率）

### 跟現有「多頭排列」是否冗餘？

- 現有多排條件只要求**今日 5MA > 10MA > 20MA > 60MA**、不包含**明日預判**
- 扣抵值是 forward-looking、補強「明天會不會繼續站 5MA 上」的維度
- **不冗餘、強烈建議加入** — 因為老師明示這是「值得這個課」的核心點 [10:32–10:35]

### 不要做的事

- ❌ 不要把扣抵值當 hard filter（會把很多老師股篩掉）
- ✅ 應該作為 score 加分項：`next_day_5ma_up_ok = True` → +10 分
- ✅ 或標記在 hit dict 給 user 看：`kou_warning: "明日扣抵不利、5MA 可能下彎"`

## 5. 改進建議（按 priority 排序）

### P0：扣抵值 5MA 預判（必加）

- **改什麼**：detect 函式內計算 `kou_5d_prev_close = bars_df.iloc[-6]["close"]`、輸出 `next_day_5ma_up_threshold`、`next_day_5ma_up_ok`
- **老師原話**：[08:44]「扣抵值是唯一可以預測的東西」、[10:32]「值得這個課」
- **影響的 case**：5/20 群創（老師明示明日 5MA 上揚 = 進場確認）；任何 close 跌破 6 日前 close 的整理股 → 標記「明日 5MA 將下彎、壓力大」
- **實作難度**：低（一行 pandas iloc）

### P1：小結構整理 hard filter（建議加）

- **改什麼**：新增 `require_small_structure: True` + `small_struct_window: 6` + `small_struct_range_pct: 6`（最近 6 日 high-low / 中位 close < 6%）
- **老師原話**：[02:08]「整理了七八天、超過原有的兩三天」、[04:08]「他現在就是要來守這個區間」
- **影響**：把「在小區間守住 5MA 的整理股」分出來、剔除「拉完一根馬上又拉一根的奔放股」
- **實作難度**：中（要先算 N 日 range）
- **note**：老師原話「七八天」、scanner cfg 從 4 改 6–8 比較貼近、但**老師沒給明確數字**、需 user 校正

### P2：N 字段位分類（建議加 label、不一定加 filter）

- **改什麼**：detect 輸出新增 `pattern` 欄位（"small_structure" / "fast_n" / "unknown"）、不必當 hard filter
- **老師原話**：[03:11–03:17] 老師明示「這次不一樣……不是要做 N 字標準形態」 = 老師會主動分類
- **影響**：給後續 sizing/holding 邏輯用（fast_n 不壓多部位、small_structure 可波段）
- **實作難度**：中

### P3：放寬月線條件（5/20 教學沒提）

- **改什麼**：考慮把月線斜率 + 月多排從 default 拿掉、改為 optional `--require-monthly`
- **理由**：5/20 教學完全沒提月線、月線是 5/28 晚課條件、兩堂課應該分開設定
- **影響**：可能讓 4 月初 V 型反彈 case 進入 strict 候選清單（backtest Recall 0% 問題）
- **實作難度**：低（已有 `--no-monthly-slope` flag、改 DEFAULT_CFG default 即可）
- **note**：但 backtest 顯示 strict 含月線 precision 48.1% > relaxed 37.5%、改前要重跑 backtest 確認

### P4：「7–8 日整理」改 consolidation_days = 7（小調）

- **改什麼**：`consolidation_days: 4 → 7`
- **老師原話**：[02:08]「整理了七八天」
- **影響**：和 P1 整合、把整理視窗從 4 拉長到 7、貼近老師明示
- **實作難度**：低（但目前這欄只是註解、需先實作 P1 才生效）

## 6. 不要改（常見誤改、但老師 5/20 沒教）

| 誤改方向 | 為何不要改 |
|---|---|
| 加「量縮量」hard filter | 老師 5/20 紅框內 K 棒確實量縮、但**整堂課沒明示「量縮 X%」門檻**。CLAUDE.md 規定「量縮 ≠ 窒息量」、避免自行發明數字。 |
| 加「拉兩根半才整理」位階條件 | [00:32]「他一根兩根兩根半跳在這個地方」是聯電 case 描述、不是普遍規則。老師沒說「進場前必須先看到兩根半起漲」。 |
| 加「股本大才小結構、股本小才快 N 字」 | [01:18–01:24]「因為它這個量體太大、然後你要這樣拉實在太累」是聯電/群創特定 case 的解釋、不是普遍 filter。 |
| 把 MA5 距離放寬到 8%+ | 老師 5/20 沒提乖離百分比、但「公回十字線」概念明示要回到 MA5 附近、距 5MA < 5% 是合理工程化、不要為了多抓而放寬。 |
| 加「漲停隔日跳空」過濾 | 5/20 教學沒提、這是 memory `feedback_entry_timing_rules` 的內容、不應寫進 swing scanner（屬不同層級）。 |

## 7. 不確定處（需 user 校正）

1. **「七八天」整理 vs cfg = 4** — 老師原話 [02:08]「整理了七八天、超過原有的兩三天」、但沒給「N 日整理 = small_structure」明確閾值。建議 user 看更多老師案例後決定（5、6、7、或 4–10 區間）。

2. **「不要低於 35.4」用 open 還是 close** — 老師原話 [08:54] 用「開盤不要低於」、嚴格扣抵值定義應該用 close、open 是盤中即時預判。實作時建議 close 為主、open 為輔助 alert。

3. **「小結構區間」range 收斂門檻** — 老師沒明示「平台 = ±X%」、是否用 ATR / 標準差 / high-low 都是工程化選擇、需 user 校正。

4. **是否將月線條件移出 default** — 5/20 沒提、5/28 有提、兩堂課優先順序需 user 決定（建議：保留月線、因為 backtest precision 顯示有效）。

## 8. 5/20 群創 case 實證對照

scanner 5/20 strict 命中 3481 群創、precision 88%（當日最高之一）。對照本課教學：

| 條件 | scanner 是否觸發 | 老師理由 |
|---|---|---|
| 多頭排列 | ✓ | 隱含（5MA 守住）|
| 距 MA5 0.1% | ✓ | 「公回十字線」[03:25] = 回 MA5 附近 |
| 距 MA10 5.9% | ✓ | — |
| 周轉率/量能 | ✓ | （老師 5/20 沒講）|
| 月線多排/斜率 | ✓ | （老師 5/20 沒講）|
| 整理日數 | ⚠️ 沒實作但實際符合 | 「整理了七八天」[02:08] |
| 扣抵值 5MA 上揚 | ❌ 沒實作但老師明示符合 | 「明天扣抵 35.4、5MA 保證上揚」[08:54] |
| N 字段位 | ❌ 沒實作但場景符合「小結構末端」 | 「他現在就是要來守這個區間」[04:08] |

**結論**：scanner 抓到 3481 是「9 條件 happen to cover」、不是「依老師教學主動偵測」。加入 P0（扣抵值）+ P1（小結構）+ P2（N 字 label）後、scanner 對 3481 的 hit 會帶有完整老師理由說明、而非只是技術 filter 通過、user 信心度會提升。
