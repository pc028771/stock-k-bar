# Mode C Scanner「高檔收斂後平台」Backtest 驗證報告

> 執行日期: 2026-05-30
> Scanner: `scripts/zhuli/entry/teacher_small_structure.py`
> DB: `~/.four_seasons/data.sqlite`
> 課程來源: 主力大全方位操盤教戰守則 5/20 N字上攻教學（群創 5:10、聯電 2:24）

---

## Executive Summary

1. **Pattern spec 本身可運作**：Scanner 每日能產生 39–106 檔候選，7 日 close range < 5% + 高檔 + 量縮的邏輯合理，無程式錯誤。
2. **老師示範的兩個主案例（群創 3481、聯電 2303）均無法被 Mode C catch**：根本原因是這兩檔在 5/15–5/21 整個觀察窗口的 7 日 close range 維持在 8–22%，遠超 5% 的平台門檻。老師 5/20 課程用這兩檔示範的是「已完成的 N 字型態回顧」，不是「進場前夕的 mode C 訊號」。
3. **7 檔擴充案例全部 miss**：3481/2303/2376/2379/2327/2317/6285 在所有掃描日均不滿足 Mode C spec，主因為 7d range 過大（8–26%）、均線排列未完整，或 MA5 扣抵方向未達標。
4. **命中數過多**：預設 spec 在 5/15 有 39 檔、5/21 達 106 檔，信噪比低，與基準市場報酬相比無顯著優勢（5/19 scan：mean +5.2% vs 基準 +9.7%）。
5. **結論：pattern 需重新校準後才能 promote。** 目前 spec 可作為初步篩選工具（輔助）但不具獨立預測力；需加入量級 / 族群過濾或調整平台定義。

## Scanner Spec 摘要

| 條件 | 設定 | 說明 |
|------|------|------|
| 高檔狀態 | close >= high_60 × 92% | 距 60 日高點不超過 8% |
| 平台收斂（hard filter）| 7 日 close range < 5% | 用 rolling window 最高/最低 close |
| MA5 扣抵向上 | close[-1] > close[-6] | 確保 MA5 明日繼續上揚 |
| 量縮 | vol_ratio_20 < 1.3 | 量低於 20 日均量 30% 內 |
| 多頭（鬆）| close > MA10 | 不要求完整多頭排列 |
| 在均線上 | close > MA20 | |

訊號分類：
- **突破前夕**：連續平台 ≥ 5 天 + MA5 扣抵🟢 + 量縮
- **已突破**：today close > max(近 N-1 日收盤)
- **高檔收斂中**：連續平台 3–5 天
- **整理中**：其他

## 跟 Mode A 比較 — Overlap / 差異

以 2026-05-20 為例：

| | 命中數 | 描述 |
|-|--------|------|
| Mode A strict | 12 | 完整多頭排列 + 月線 + 周轉率 + 乖離 |
| Mode A relaxed | 56 | 放寬月線 + 周轉率 |
| Mode C | 87 | 高檔 + 平台收斂 + 量縮 |
| A strict ∩ C | 0 | **完全不重疊** |
| A relaxed ∩ C | 1 | 僅 6509 一檔 |

**差異分析**：
- Mode A 要求完整多頭排列（MA5>MA10>MA20>MA60），自然過濾掉平台整理中均線尚未完整排列的股票
- Mode C 只要求 close > MA10/MA20（鬆多頭），會 catch 到大量「橫盤中、均線還在整理」的個股
- 兩者**互補而非替代**：Mode A 抓已完成排列的強多頭，Mode C 理論上抓排列前夕；但實際上 Mode C 的候選質量需二層過濾

## Sensitivity 建議

若要 catch 到更多「老師示範型」個股，以下為各門檻的候選數變化（以 5/20 為例）：

| range 門檻 | 5/20 候選數 | 3481 catch? | 2303 catch? |
|------------|-----------|------------|------------|
| < 5% | 87 | ❌ | ❌ |
| < 7% | 110 | ❌ | ❌ |
| < 8% | 116 | ❌ | ❌ |
| < 10% | 132 | ❌ | ❌ |

**結論**：即使把 range 放寬至 10%，3481/2303 仍無法被 Mode C catch（因為即使 range 通過，其他條件如 vol 或 MA5 kou 方向仍不符合）。根本原因是這兩檔在觀察窗口的型態屬於「拉升中」而非「拉升後平台」。

若要減少雜訊（候選數 < 20），建議加入：
1. **成交量門檻**：vol_lots > 3,000 張（過濾小型股）
2. **多頭排列分數**：ma5 > ma10 加分（不硬過濾，但排序優先）
3. **族群過濾**：結合老師當日提及族群做 cross-filter

## 結論：Pattern 是否 Valid？要不要 Promote 成 Production？

**現狀：建議作為輔助工具，暫不 promote 為主要 scanner。**

| 評估項目 | 結果 |
|---------|------|
| 老師示範案例 catch 率 | 0/7（所有擴充案例均 miss）|
| 後 10 日報酬 vs 基準 | 跑輸（-4.5% to -7.5% vs 基準）|
| 信噪比（候選數） | 過寬（39–106 檔）|
| 程式正確性 | ✅ 無 bug，邏輯一致 |
| 課程依據 | ✅ 老師 5/20 親口示範 |

**Pattern 假設的問題**：老師示範的群創/聯電是「N 字型中段整理」（漲了 40–80% 後的短暫整理），這段整理的 range 遠超 5%；Mode C 的 5% range 門檻更適合「已在高位盤整很久（7+ 天、range 緊）的橫盤股」，這類股票與老師示範的例子不同。

**建議後續方向**：
1. 改為「比較日期前一段上漲後的整理」定義（需找出「近 N 天有 +10%+，之後進入 M 天整理」的序列）
2. 或接受 Mode C = 純橫盤股過濾器，搭配老師族群方向二層過濾使用
3. 跑更長時間窗口（6/2–6/5）後再評估是否有改善空間
