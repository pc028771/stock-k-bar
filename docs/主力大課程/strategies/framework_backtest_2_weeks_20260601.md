## 漏抓的漲停股

整個 2 週期間市場共有 **188 檔個股** 從開盤到收盤漲幅 ≥ +9.5%。

**核心問題：scanner 候選名單與市場漲停股 = 0 重疊**

| 觀察 | 細節 |
|---|---|
| 漲停股特徵 | 多為小型散戶題材股 (6434/6819/6572/7868 等)、非老師族群 |
| Scanner 候選族群 | 主要為老師核心族群：ABF載板、機器人、矽晶圓、成熟製程 |
| 新 framework 抓到漲停股 | 0 個 |
| 舊 framework 抓到漲停股 | 0 個 |
| 根本原因 | Scanner 的設計是找「波段整理後起漲」型態，而非「當日暴漲」型態 |

**觀察名單 (太遠未進場) vs 後續表現：**

|---|---|---|---|

→ **不能單純說漏抓，因為這些標的本就超出距 MA10 ≤10% 的進場條件**，進場風險高。

→ 5/27 籌碼資料缺失，即使有籌碼也難以防守這種「開高走低」型態，需加入技術面的「開盤型態確認」層。

## 結論

### 新 framework 是否系統性比舊好？

**有一定統計支持，但樣本太少 (僅 4 個有效差異點，且其中 1 個對新不利)。**

- 最大貢獻來自單日 (5/22，差距 +48.9K)，屬於「台達電被排除」的一次事件
- 若 5/22 那天台達電有籌碼資料但確認強，新 framework 可能反而不如舊

**量化結論：**  
- 新 > 舊：+41,400 (約 +44% PnL 提升)  
- 但兩者都是虧損，core issue 不在籌碼層而在入場時機  

### 還有什麼盲點

| 盲點 | 重要性 | 說明 |
|---|---|---|
| **Broker tier 1 分點** (站前哥/館前哥/三爹) | 🔴 高 | 老師指名 broker 買盤 = 最強訊號，完全未整合 |
| **融資餘額變化** | 🔴 高 | 融資大增 = 倒貨警示，8046/6139 沒有此檢查 |
| **開盤型態確認** | 🟡 中 | 開高 > +3% 後 5 分鐘走弱 → 不進；當前只看開盤價 vs 前收，沒看盤中 |
| **老師當日 Line 修正** | 🟡 中 | 老師盤後/盤中的族群切換 (如 5/27-5/28 矽晶圓接棒) 無法被靜態 scanner 捕捉 |
| **籌碼資料完整性** | 🟡 中 | 50%+ signal 日籌碼資料缺失 (特別是小型股)，大幅限制新 framework 的差異化能力 |
| **漲停股** | 🟢 低 | 這些股票根本不在 scanner universe，是另一種策略需求 |

### 推薦 Production 改動

1. **優先 (短期)：** 補齊 broker tier 1 資料 (`chip_broker.py`)，加入站前哥/館前哥分點出現 → 加分；同時加融資餘額篩選
2. **中期：** 在 scanner 結果上加「老師今日 Line 族群是否匹配」的 tag 系統
3. **長期：** 把 5/28 那種「開高後 5min 走弱」型態做成盤中觸發的 early-exit signal
4. **不建議改的：** scanner 的技術型態邏輯是正確的 (ABF/矽晶圓/機器人都是老師族群)；問題在過濾層不足，不是 universe 本身
