**核心結論**：Stage SOP（不自動停損 + Stage 2 情境加碼）贏 v3 約 **+366,150 NTD**，主要來自 8 個「情境B回撤反彈」cases——這些 cases 用 v3 會全部被 -3% 停損洗掉，合計損失 -81,650 NTD；Stage SOP 拿到 +229,200 NTD，差距 +310,850 NTD。

**唯一反例**：4979 華星光（情境C失敗），Stage SOP 比 v3 多損 -49,000 NTD，原因是 SOP 等結構確認失敗才出場（-17%），而 v3 在 -3% 就停損。

## 2. User Mental Model 解讀

User 的真實操作 SOP 核心在於**不用固定百分比停損，而是等結構訊號**：

```
時刻 0（訊號日）: 週報發布 → 課程 SOP 篩選
  ↓ T+1 open
Stage 1: 試單 1 張（探路，不預設走勢）
  ↓ 追蹤後續走勢
  情境A — 直接強勢延續（連紅+量加）→ Stage 2 追高加碼
  情境B — 先回撤然後反彈（守前低+紅K+量）→ Stage 2 低接加碼
  情境C — 一直跌、跌破結構底、不反彈 → 出 Stage 1，只虧 1 張

出場：
  ① 達標 +30%（題材股實務）
  ② 跌破 MA10 + 有獲利 → 停利出場
  ③ 結構失敗（連 5 天破前低 + 跌幅 > -10%）
  ④ 60 天時間到
```

### 課程依據

- Stage 1 試水：`feedback_scaling_in_entry.md`（老師「先試 1 張」）
- Stage 2 情境A/B：Ch5-3 5K SOP（5/19 老師當沖實戰）+ 5/1 復盤課強勢攻擊 / 反彈段判斷
- 結構底跌破出清：老師「結構底跌破出清」（5/19 課程）
- 加碼 +10% 門檻規則：老師 5/29 親口確認（本 backtest 模擬 Stage 2 加碼未強制要求 +10%，因為 Stage 2 屬於「已有 Stage 1 存活」情境下的加碼，而非從零加碼）

## 6. 即時 Stage Helper 工具設計需求

實際交易時需要一個「Stage 2 觸發偵測器」，設計規格如下：

### 情境 A 偵測（強勢追高加碼）
```
輸入: ticker, entry_price, 最近 3 日 K 棒
觸發條件:
  1. 連 2-3 日收紅
  2. 當日收盤 > 前日收盤（上漲延續）
  3. 量能 ≥ 前日（不縮量）
  4. 收盤高於 entry_price × 1.05+（已有獲利緩衝）
輸出: 加碼建議 + 建議進場點（當日收盤）
```

### 情境 B 偵測（反彈低接加碼）
```
輸入: ticker, entry_price, 最近 N 日 K 棒, 是否有回撤 flag
觸發條件:
  1. had_pullback = True（從 entry 跌過 -2%）
  2. 當日收紅（底部反彈 K）
  3. 當日低點守前日低（不破低）
  4. 量能 ≥ 前日（反彈量確認）
輸出: 加碼建議 + 建議進場點（反彈日收盤）
```

### 情境 C 偵測（結構失敗確認）
```
輸入: ticker, entry_price, 最近 N 日 K 棒
觸發條件:
  1. 連 5 日低點持續破前低（趨勢向下）
  2. 總跌幅 < -10%
  3. 無量縮止跌訊號
輸出: 出清警報 + 建議出場點（即時收盤）
```

### 整合介面
- 每日收盤後自動掃描所有開倉部位
- 對符合 A/B/C 條件者推送通知
- 標記原因（哪個條件觸發）

## 7. 啟碁（4926）6/2 重新解讀

> 本節根據 Stage SOP v4 框架重新評估，不沿用 v3 的 -3% 停損思維。

**6/2 早盤狀況**（根據 user 掌握資訊）：
- 啟碁為老師點名題材股（WiFi 7 / 網通族群）
- 若於訊號日進場，T+1 可能遇到早盤強勢

**Stage SOP v4 評估框架**：
1. **Stage 1 進場時機**：等開盤後 9:10（前 10 分鐘不切入，雙錨停損規則）
2. **雙錨停損**：取「開盤價 / 昨收 / 第一根5分K低點」三者最高為停損參考
3. **若早盤前 5 分鐘漲幅 > 5%** → 今天 skip，不追（短線進場紀律三鐵）
4. **情境判斷**：
   - 若早盤強勢延續（連紅 + 量加）→ 等確認後 Stage 2 追高
   - 若早盤衝高回落 → 等轉折訊號再評估 Stage 2
5. **不用 -3% 自動停損**：等結構失敗訊號（連跌 5 天 + 跌幅 > -10%）才出

**關鍵紀律**：啟碁為警示族群（PCB/連接器相關？）→ 若屬警示族群，依規則縮半 sizing（Stage 1 = 0.5 張概念，實務取整 = 1 張但降低總曝險）。

## 8. 模型限制與聲明

1. **樣本數**：20 cases（低於 30 的警示門檻），結論方向正確但統計信心度有限
2. **日 K 模擬**：Stage 2 精確進場點（反彈日收盤）在實際交易中需要盤中判斷，日 K 模擬為近似值
3. **「未加碼」cases**：11 個 A 情境中有 5 個未觸發強勢加碼條件（連 3 日紅 + 量），可能錯過部分加碼機會；若放寬條件（連 2 日紅）樣本會增加，但 Stage 2 訊號品質下降
4. **4979 華星光反例**：SOP 比 v3 多損 -49,000 NTD，原因是等待結構確認的成本。在 CPO 族群行情反轉時，SOP 的保持耐心反而放大了損失——這是 Stage SOP 的已知弱點：**行情直接向下不反彈的情況**
5. **費用未計入**：手續費 0.0399%/side + 證交稅 0.3%，約影響 ±3,000-5,000 NTD/case
