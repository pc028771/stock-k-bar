# Framework Backtest — 兩週回測 v2 (嚴格 DB 驗證版)

> **重跑原因**: v1 (Sonnet 跑版) 被 Opus 抓到 3 處嚴重數據錯誤：
> 1. 5/26 6282 宣稱 67.0→72.0 (+7.5%)，DB 實際 63.6→62.4 (-1.89%)
> 2. 5/27 8046/6139/2426 宣稱「籌碼全無」，DB 實際全部有資料
> 3. 5/21 2308 宣稱「無籌碼」，DB 實際有
>
> **v2 規範**: 每個數字皆有 SQL query 結果佐證，任何「缺資料」宣稱必須先見到 `rows = 0`。

---

## 回測設定

| 項目 | 設定 |
|---|---|
| 期間 | 2026-05-19 ~ 2026-05-30 |
| DB | `~/.four_seasons/data.sqlite` (read-only) |
| 表 | `standard_daily_bar`, `institutional_investors` |
| Sizing | $300,000/檔，N 張 = floor($300,000 / open / 1,000) |
| 進場 | T 日 open |
| 出場 | T 日 close（當沖簡化版） |
| 紅線 #9 | (T_open - T-1_close) / T-1_close > 5% → skip、用 backup 第 4 檔 |
| 舊 framework | scanner Tier-A/B 排名前 3（純技術） |
| 新 framework | 同上 + T-1 籌碼篩選：外資5d ≥ +2000 張 OR 投信5d ≥ +200 張，否則換 backup |
| 籌碼 5d | T-1 往前最近 5 個交易日 SUM |

## 結論

1. **籌碼篩選節省了 +79,900 元**，來源分拆：
   - 05-22 避開 2308 虧損：+30,000
   - 05-25 避開 1303 小虧：+900
   - 05-28 避開 8046 虧損 -55,000、但犧牲 2426 獲利 +6,000：**淨 +49,000**

2. **本次 scanner 覆蓋率低**：9 個交易日只有 3 日有 scanner 資料 + Tier-A/B 候選，導致回測樣本偏小（n=8 舊 / n=4 新），結論參考性有限。

3. **新 framework 仍虧損**（-34,400），主因是 6139 亞翔 05-28 當日大跌（+3,549 投信僅反映主動買，外資賣超 -2,407，結果收黑 -5.4%）。籌碼篩選無法防範當日出貨行為。

4. **數據說明**：6282 在 05-25 跳空 +4.88%（prev_close=57.4, open=60.2），未觸發 >5% 紅線，可進場。
