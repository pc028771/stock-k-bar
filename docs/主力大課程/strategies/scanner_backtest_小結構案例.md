## 4. 失敗案例分析

### 廣達 (2382) — MA 結構不完整（設計性排除）
- **根因**：ma10 < ma20 整個觀察窗口，多頭排列 hard filter 未過
- **解讀**：廣達屬於「均線尚未完整整理好」，技術上不在打擊區。老師說「廣達比緯創好」是戰略（好公司、跟對族群），不代表技術形態準備好
- **改進可能**：加「ma10 即將穿越 ma20（黃金交叉前 1-2 日）」pre-alert；但這超出課程框架

### 啟碁 (6285) — 整理時跌破 MA5（打擊區判定正確）
- **根因**：整理期間 close < MA5（-2.5% 最深），師判「通過考驗」是事後確認，不是事前
- **解讀**：teacher_swing 正確排除。起漲前 close < MA5 = 「還在考驗中」；5/20 收 279（dist_ma5=+1.2%）才進打擊區，但那已是第 3 天漲（dist_ma10=+3.6%）
- **改進可能**：加「MA5 上揚但 close 暫時跌破 MA5 = 觀察清單（非進場）」flag

### 康舒 (6282) — 底部反轉型，不是均線上整理型
- **根因**：6282 是缺口 + 整理 + 再缺口的型態，不是「ma5>ma10>ma20>ma60 多頭整理末端」
- **解讀**：teacher_swing vs small_structure.py 的功能分工問題，非 teacher_swing 的 bug
- **改進可能**：兩個 scanner 互補使用（teacher_swing 主流、small_structure 底部反轉）

## 5. Scanner 改進建議（基於失敗案例）

### 建議 1：距 MA10 降格為評分（最重要修改）

**現況**：dist_ma10 < 8% 為 hard filter（strict）/ < 12%（relaxed）
**問題**：3481 在 5/19（D-2）dist_ma10=**10.1%** → 被嚴格模式過濾，但兩天後漲停

**新框架**（依 User 5/29 指示）：

| dist_ma10 範圍 | 評分 | 操作建議 |
|---|---|---|
| 0% < dist ≤ 8% | ⭐⭐⭐ | 打擊區最佳 |
| 8% < dist ≤ 15% | ⭐⭐ | 已啟動中段（仍 valid，但 sizing 保守）|
| > 15% | ⭐ | 末段提示，mention 但不主推 |

**Hard filter 改為**：
1. close ≥ MA5（binary）
2. MA5 > MA10 > MA20 > MA60（多頭排列）
3. MA5 slope > 0（上揚）
4. 量（周轉率或 5 日量峰）

**影響評估**：在本次 8 個案例中，3481 D-2 從 miss → ⭐⭐ catch，實際多增 1 個提前 catch。

---

### 建議 2：新增「MA5 上揚但 close 暫低於 MA5」= Watchlist Flag

**現況**：close < MA5 直接排除
**問題**：6285 在 D-2 close=-2.5% 低於 MA5 但 MA5 持續上揚（slope=+0.95%）

**新邏輯（Watchlist, 不進場）**：
```
if (close < ma5) AND (ma5 > ma10 > ma20 > ma60) AND (ma5_slope > 0.5%):
    flag = "MA5_上揚_收盤未達"  → 加入「等待轉折」清單
    entry_trigger = "close ≥ ma5 的當日"
```

此 flag 在 6285 D-2（5/15）即可出現，5/18 收盤站上 MA5 → 進場訊號。

---

### 建議 3：大型股周轉率降格為 Rank（解決鴻海類型 miss）

**現況**：周轉率 < 1.3% strict / < 0.5% relaxed = hard fail
**問題**：鴻海 2317 周轉率天然只有 0.88%，但起漲時量已是 76,000 張（5/27）

**新邏輯**：
```
if 成交量 ≥ 50,000 張:
    turnover_check = "量足夠（大型股豁免）" → pass
    rank 降一星（⭐⭐⭐ → ⭐⭐）
else:
    turnover_pct = 正常計算
```

注：code 已有類似邏輯（`vol_lots_1d >= 50_000` fallback），但 2317 5/26 因周轉率 0.40% < 0.5% 被 relaxed 排除；問題出在 shares_issued 資料滯後（最新只到 5/19）。
**實際解法**：`shares_issued` 缺更新時，直接用 50,000 張 fallback（不依賴滯後資料）。

## 6. 觀察清單 Candidate（「等待轉折」pre-launch）

根據 backtest 找出兩類 pre-launch 訊號：

### Type A：MA5 上揚但 close 暫時低於 MA5
```python
# 觀察清單 Flag（不進場）
if (close < ma5) and (ma5 > ma10 > ma20 > ma60) and (ma5_slope > 0):
    watchlist_type = "MA5_整理中"
    # 等 close >= ma5 為進場觸發
```
**代表案例**：6285 啟碁 5/14-5/15

### Type B：多頭排列完整但距 MA10 8-15%（過熱觀察）
```python
# 過熱但仍 valid
if bullish_array and (8 < dist_ma10 <= 15):
    watchlist_type = "中段啟動"
    # sizing 保守、不全倉
```
**代表案例**：3481 群創 5/19

### 偽碼整合

```python
def classify_ticker(close, ma5, ma10, ma20, ma60, ma5_slope, dist_ma10):
    if not (ma5 > ma10 > ma20 > ma60):  # 未多頭排列
        if ma5 > ma10 and abs(ma10 - ma20) / ma20 < 0.03:  # 即將黃金交叉
            return "PRE_ALERT"  # 廣達型
        return "NOT_READY"
    if close < ma5:
        if ma5_slope > 0:
            return "WATCHLIST_MA5_UNDER"  # 啟碁型
        return "NOT_READY"
    dist = (close - ma10) / ma10 * 100
    if dist <= 8:   return "⭐⭐⭐"  # 打擊區
    if dist <= 15:  return "⭐⭐"    # 中段
    return "⭐"                       # 末段
```
