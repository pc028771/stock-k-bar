## 一、4 個出場 Detector 設計

### 1. 🌂 掀傘 (Open_umbrella)

**課程來源**: `docs/主力大課程/主力大_5分K規則_spec審查_20260528.md` L48-51

**條件** (持倉中、賺中):
- 連續 2-3 根 5K / 日 K 不創新高
- 量縮 (vol_ratio < 0.7 vs 前 10 根均)
- 無連續紅K

→ 主動全出，精神「主力收手了、不等被動停損」

**實作位置**: `scripts/zhuli/exit/detectors.py` → `check_umbrella_exit()` (5K 版) / `check_umbrella_exit_daily()` (日線 backtest 版)

---

### 2. 🦘 高檔長黑 K (High_long_black)

**課程來源**: K線力量判斷入門 單一K線(7) 高檔區域的長黑K

**3 種意義 (≥2 種 = 攻擊結束)**:
- M1 缺口回補: 收盤 < 最近 gap-up 下緣
- M2 包覆創高紅K: 前 1 根紅 K 創 60d 高 + 今天大黑 K 嚴格包覆
- M3 吃下前 5 根: 收盤 < min(前 5 收)

**高檔過濾**: 過去 60d 高/低 ratio ≥ 1.3

**實作位置**: `scripts/zhuli/exit/detectors.py` → `check_high_long_black()`
**也可 import**: `scripts/kline/exit/high_long_black.py` (DataFrame 批量版)

---

### 3. 💰 分批停利里程碑 (Profit_taking_milestone)

**課程來源**: memory `feedback_swing_stop_profit_rules` + 老師教法

**觸發里程碑**:
- +10% → 通知「考慮鎖利 1/3」
- +20% → 通知「再鎖 1/3」
- +30% → 通知「最後 1/3 看結構」

每個里程碑只提示 1 次、不重複。Monitor 透過 `_profit_milestones_state` dict 跨循環維護。

**實作位置**: `scripts/zhuli/exit/detectors.py` → `check_profit_milestone()`

---

### 4. 📉 隔日跳空大跌 (Gap_down_alert)

**課程來源**: 老師「跳空缺口回補失敗」+ memory `feedback_dump_signal`

**觸發條件** (9:00-9:10 開盤後立即評估):
- 跳空 ≤ -5%: 緊急 — 立即出
- 跳空 -3% ~ -5%: 警示 — 注意
- 跳空 > -3%: 正常 — 不通知

**實作位置**: `scripts/zhuli/exit/detectors.py` → `check_gap_down_emergency()`

## 三、重要案例分析

### 案例 1: 3016 嘉晶 -$21,388 (最大損失)

**情況**: 6/2 試撮 FOMO 進場 @134，同日收 118.5 = **-21,388 一天**。

**急殺 Detector 分析**: 未觸發。原因: 嘉晶是 9:00-9:05 進場 = **違紀 (紅線 #8)**，不符合正常「持倉中隔日評估」的使用情境。急殺 Detector 無法救違規進場。

**根本問題**: 不是缺出場工具，而是**根本不該進場**。
- 試撮漲停 ≠ 真實開盤 (memory `feedback_premarket_match_fomo_trap`)
- 必須等 9:10 後確認

### 案例 2: 3481 星宇航 +$92,909 (最大贏家)

**分批停利里程碑觸發記錄**:
- 05-25 → +10% 里程碑 ($49.1) → 提示鎖 1/3 (1,666 股)，單次 PNL +13,856
- 05-27 → +20% 里程碑 ($50.2) → 提示再鎖 1/3，單次 PNL +15,682
- 06-01 → +30% 里程碑 ($56.1) → 提示守剩 1/3，單次 PNL +25,478

**若執行分批停利**: 3 次各鎖 1/3 → 早期鎖利合計 ~+55,016，保留 1/3 繼續持有。
比全倉持有到 6/3 少賺 ~+37,893 (92,909 - 55,016)，但**消除了回檔風險**。

**結論**: 分批停利對趨勢股最有效，保護已賺利潤、降低被一把清光的風險。

### 案例 3: 2885 元大金 (掀傘唯一觸發)

**掀傘觸發**: 06-02 收 $59.6，連 3 日 (05-29/06-01/06-02) 不創前波高 $60.2 + 量縮 ×0.67。

**對比**:
- 若 06-02 掀傘出: PNL +13,743
- User 仍持有到 06-03: 收 $63.7，PNL +54,603 (+40,860)

**結論**: 掀傘在 2885 案例是**誤出**。06-03 大漲 +7%，掀傘過早出場少賺 $40,860。

**但更重要的學習**: 掀傘是**5K 版盤中工具**、而非日線。日線版誤觸發原因是震盪盤的量縮+橫盤本就常見。實際 monitor 中，掀傘應**搭配 5K 實時看**，日線回測版有過度敏感問題。

### 案例 4: 2476 鉅祥 / 3149 正達 (分批停利提前鎖利)

| 標的 | 分批停利觸發 | 實際出場 | 若執行 1/3 vs 全持 |
|------|------------|----------|-------------------|
| 2476 | 05-26 +10% @131 | 05-28 $121.5 (-6k) | 鎖 1/3 @131 → 部分獲利 +10k，剩 2/3 繼續回落 |
| 3149 | 05-26 +10% @68 | 05-28 $67.2 (+23k) | 鎖 1/3 @68 → 部分早鎖，剩 2/3 出 @67.2 |

2476 的案例說明分批停利有實際幫助 — 原本最終 -6,395，若 05-26 +10% 時鎖 1/3，可救回 ~10,000。

## 六、Detector 有效性評估

| Detector | 評估 | 最佳使用情境 |
|----------|------|-------------|
| 🌂 掀傘 | **中** — 日線版容易誤觸 (震盪盤橫盤量縮)，5K 版較準。需搭配課程「賺中」條件過濾。| 強勢股衝高後觀察是否主力收手 |
| 🦘 高檔長黑 | **嚴格** — M2 包覆要求嚴格，此期間零觸發。條件合理，等待高檔 + 強力反轉才觸發 | 熱門股創高後強力反轉 |
| 💰 分批停利 | **最實用** — 4 次 +10% 觸發全部有意義，幫助保護波段利潤 | 所有 core 持倉 |
| 📉 隔日急殺 | **保護型** — 此期間無 ≥-3% 跳空 (市場偏多)，平常市場有效 | 持倉遇系統性下殺、主力出貨日 |

**最值得啟用**: 分批停利 (所有持倉)、急殺預警 (持倉保護)
**需要更多樣本**: 掀傘 (需 5K 即時驗證)、高檔長黑 (需等更多高檔攻擊後反轉案例)
