## 3. 跨策略量化 Gap 統整

以下是在多個策略中重複出現的「定義不清」問題，scanner 實作時必須做出工程決策。

### 3.1 「大量」/ 「爆量」門檻（影響策略：A/H/L/D/G）

**問題：** 課程設計刻意不給絕對倍數，所有「大量」均為「與前一根/前一波比較」的相對概念。`pdf_extracted_parameters.md` 已確認 PDF 也無絕對門檻。

**現況：** 既有 POC 腳本（`scripts/zhuli_large_volume_threshold_poc.py`）針對 22 個案例反推，初步結果 P25=3.8、Median=4.3（相對於 MA20 的倍數）。

**工程決策選項：**
- 方案 A：`LARGE_VOL_RATIO = MA(volume, 20) * N`，N 為 CLI 可調參數，不硬編碼
- 方案 B：各策略獨立設定自己的 N 預設值（窒息量策略已有 10% 最大量的精確定義，不受此 gap 影響）

**建議：** 採方案 A，POC 報告中的 3.8-4.3 作為初始預設值建議，但不寫死為「課程標準」。

---

### 3.2 「急漲段」如何識別（影響策略：E 布林回測）

**問題：** 形態四進場條件要求「量縮 ≤ 急漲段大量紅K 的 1/5」，但「急漲段大量紅K」是哪一根？

**現況：** 最合理的錨點是「形態三觸發日（收盤站上布林上軌的那根 K棒）的量」，但課程未明說，僅口語描述「急漲後」。

**工程決策選項：**
- 選項 A：直接用「形態三觸發 K 棒的量」作為錨點（概念上最符合老師描述）
- 選項 B：用「回測前 N 日內量最大的那根紅K」作為錨點（更保守）

**建議：** 選項 A，在 code comment 中標明推理邏輯；如有 ch4-2 末段截圖 vision 補充可再驗證。

---

### 3.3 「兩高點連線」演算法（影響策略：C 反轉形態）

**問題：** 反轉形態要求「紅K 底部站上下降趨勢線（兩高點連線）」，但哪兩個高點？老師用肉眼判斷，無算法定義。

**現況：** `strategy-readiness.md` 已提出完整 POC 計畫（`scripts/zhuli_swing_high_viewer_poc.py` 已存在），但尚未跑案例校準。

**工程決策：** 此 gap 是 Phase 3 的唯一真正 blocker。`scipy.signal.find_peaks` + rolling window + 人工 ground truth 校準是唯一可行路徑。

---

### 3.4 「剛上榜」的 N 天定義（影響策略：I 投信跟單、J 投信首買）

**問題：** 投信跟單要求「剛上榜（前幾天未上榜過）」，未定義 N。投信首買要求「前 2-3 個月無投信買超」，給了時間範圍但未精確到日數。

**現況：** 兩個策略的 vision stub 均高（76/82），補完後可能有細節補充，但可能也沒有。

**工程決策：** 投信跟單「剛上榜」預設 N=10（前 10 個交易日無上榜）；投信首買空白期預設 60 天（可調至 90 天），兩者均設為 CLI 可調參數。

---

### 3.5 「距缺口附近」距離（影響策略：K 缺口輔助）

**問題：** 多方缺口進場要求「量縮回測缺口附近，未跌破缺口低點」，「附近」距離未量化。

**現況：** 課程無定義，為實作上的自由度。

**工程決策：** 預設 ≤ 3%（即股價在缺口低點至缺口低點×1.03 之間），設為可調參數。
