## 文章主旨（摘要）

尼克全方位培訓班第一篇技術面教學文章，主題為「必須上車的標準買點長相」。文章以實際範例為核心，以 4/23 前後的個股 K 棒圖說明「三天內必漲的長相」是什麼結構特徵。重點是「系統化教學第一步：強化買點」，文章聲明範例會持續增加。內容架構為圖示驅動（幾乎全部是 K 棒截圖），文字極少，讓讀者自行判讀並留言回饋。

**核心觀念**：符合特定形態結構的股票（MA 多頭排列 + 量價協調 + 近期攻擊後整理），在整理完成的當下位置就是買點，且預期三天內會起漲。

## Scanner 漏掉的 Pattern 清單

### 1. 「低位趨勢起漲初段」— 現有 scanner 最大空白

**問題**：2855 統一證（圖1）是從底部起漲初段（不是高位整理），價格剛從 22 漲到 34，MA 剛開始多頭排列。這種「起漲第一到第三天」的形態，`small_structure` 要求 `prior_attack >= 10%` + 已在高位，因此**不會觸發**。

**Pattern 特徵**（從圖1提取）：
- MA5/MA10/MA20 剛完成黃金交叉（從零散轉為多頭排列）
- 量從縮量轉為相對放量（但非暴量）
- 收盤接近當日高點（紅K收最高）
- 股價仍在低位（距 60 日高點可能僅 +5~15%）

**建議補進 scanner**：
```python
# 建議新增: low_level_breakout (低位多頭排列起漲初段)
# 條件：
# 1. MA5 > MA10 > MA20（剛完成多頭排列，過去 3 天才剛交叉）
# 2. 距 MA20 漲幅 ≤ 20%（仍在低位，非高位整理）
# 3. 量比 1.5x~4x（相對放量但非暴量）
# 4. 紅K收最高（收盤在高低差 80% 以上）
# 5. MA20 向上（MA20 斜率 > 0，確認趨勢建立）
```

---

### 2. 「完美 MA 多頭排列 + 全 MA 向上」— 現有條件不完整

**問題**：圖4(2467) 展示 MA5 > MA10 > MA20 > MA60 **全部向上**（▲標記），而現有 `contrarian_strong_scanner` 只看 MA5 > MA10（+1分），沒有要求 MA20 和 MA60 方向。

**Pattern 特徵**：
- MA5, MA10, MA20, MA60 均為 ▲（向上）
- 各 MA 之間無黏合（完整扇形展開）
- 代表多頭趨勢極為健康，接近「最強多頭」

**建議補進 scanner**：
```python
# 在 contrarian_strong_scanner 加分項中增加：
# +1 MA20 向上（ma20_will_rise == True 或 ma20 斜率 > 0）
# +1 MA60 向上（ma60 今日 > 昨日）
# 或合併為 +2「全 MA 多頭排列」
```

---

### 3. 「量縮整理後放量攻擊日」— 需要量的方向性確認

**問題**：三張圖（8996, 3651, 2467）都在展示「量縮整理後，放量突破/起漲」的那一天。現有 scanner 有量比條件，但沒有捕捉到「從量縮趨勢轉為放量」這個**轉折點**本身。

現有量條件（`contrarian_strong_scanner`）：
- 量比 ≥ 1.5x vs 過去 5 日均 (+2)
- 量比 1.0~1.5x (+1)

但這個條件單看當日量，沒有確認「前 3-5 日是縮量」這個背景。

**建議補進 scanner**：
```python
# 在量比評分中增加「量縮後放量」加成：
# 條件：今日量比 ≥ 1.5x AND 前 3 日均量比 < 1.0（窒息後放量）
# 加分：+1 on top of 現有量比分數
# 或作為獨立 filter 在 small_structure 中加嚴
```

---

### 4. 「趨勢中繼整理—不破前高 -5%」— small_structure 的 high_holding 門檻偏寬

**問題**：`small_structure` 的條件 6 是 `c / high_10d >= 0.85`（收盤在 10 日高點 85% 以上）。但從圖2(8996) 和圖3(3651) 來看，這些標的的整理幅度更淺（通常 -3% 到 -8% 回調），亦即老師展示的買點更接近 10 日高點。

**建議調整**：
- 考慮將 `high_holding` 條件從 85% 提高到 90%（更靠近高點才是老師展示的買點）
- 或加一個「近 3 日回落不超過 5%」的條件

---

### 5. 「長時間軸確認（月線/週線 MA 向上）」— 目前 scanner 只看日線

**問題**：圖5(2467 長視角) 刻意展示同一檔的更長時間軸，暗示老師買點判斷需要「日線 + 週線/月線確認」。但現有所有 scanner 都只用日線 MA（MA5/10/20/60）。

**建議補進 scanner**：
```python
# 加入週線確認 flag（可選）：
# 週線 MA5 向上 = 對應日線 MA25 向上
# 週線 MA10 向上 = 對應日線 MA50 向上
# 作為 +1 加分項（非必要條件）
```

## 補進 Scanner 建議（優先順序）

1. **最高優先**：`contrarian_strong_scanner` 增加「完美多頭排列加分」（MA20▲ +1, MA60▲ +1）— 改動小，立竿見影
2. **中優先**：加入「量縮後放量」組合條件到 `small_structure` 或 contrarian 加分系統
3. **中優先**：新增獨立 scanner `low_level_breakout`（低位起漲初段）— 填補 shakeout_strong 和 small_structure 之間的空白
4. **低優先**：調整 `small_structure` 的 `high_holding` 門檻 85% → 90%
5. **低優先**：週線 MA 向上作為 contrarian 加分項（需要先確認 DB 有週線資料）
