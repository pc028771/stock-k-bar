# 盤中執行 SOP（Phase 3）

> 整合 Phase 1+2 工具 + 主力大/K 線力量課程明示規則 + Ch2 通用框架
> ⚠️ **CLAUDE.md 紅線**：本 SOP **不指定**具體進場價、停損價、倉位比例、盤中時機 — 這些課程沒明說的，保留給 user 自己依個股結構判斷

---

## 一日交易節奏（5 階段）

| 階段 | 時段 | 主任務 |
|---|---|---|
| 1. 盤前 | 8:30~9:00 | 回顧昨日訊號、確認大盤狀況、檢視觀察清單 |
| 2. 開盤 | 9:00~9:30 | 對照昨夜推測 vs 開盤實況 |
| 3. 盤中 | 9:30~13:00 | 持股監控（M 收高開低 警示）、觀察清單動能追蹤 |
| 4. 尾盤 | 13:00~13:30 | G 隔日沖進場時點 / 持股收盤決策 |
| 5. 盤後 | 13:30+ | 收盤確認、跑 scanner 更新明日清單 |

---

## 1️⃣ 盤前（8:30~9:00）

### 必做：跑 watchlist 確認狀態

```bash
python scripts/zhuli/watchlist_intraday.py
```

**看什麼：**
- **共識分數**（30D 過去 multi-scanner 命中）
  - 🟢🟢 score 3+：歷史 PF > 2.10，可優先關注
  - 🟡 score 1：baseline 訊號
  - ⚪ score 0：無 multi-scanner 確認，需單獨評估
- **扣抵預判燈板**
  - 🔴 MA 將下彎 = **早期出場警示**（比課程「趨勢特徵消失」早 1-2 天）
  - 🟡 MA 臨界：1-2 天內可能轉折
- **個股觀察指標** ✓/✗ 課程框架檢核

### 必做：大盤面確認（課程 Ch6-1）

依 §G 隔日沖 spec：
- 加權指數：量增紅K + 收盤 > 5ma
- OTC 指數：量增紅K + 收盤 > 5ma
- **兩者皆符** = 大盤友善，當沖/隔日沖可考慮
- **任一不符** = 大盤偏空，僅保留中長波段策略

---

## 2️⃣ 開盤（9:00~9:30）

### 持股檢核：M 收高開低 出場警示

對每檔持股檢查（**用昨日為基準**）：
- 昨日是否「收最高 + 出量」？
- 今日開盤是否 ≤ 昨收？

→ 若 **M bearish_exit + 漲停板平盤警示同時成立** = 主力試空雙重警示
- 課程紅線：**收盤確認**，盤中震盪非依據
- 警示後等收盤再決定

### 觀察清單：開盤跳空判讀

依課程 Ch2-3：
- **跳空向上**（gap_up）：今 low > 昨 high → 多方缺口，缺口低為支撐
- **跳空向下**（gap_down）：今 high < 昨 low → 空方缺口，缺口高為壓力

---

## 3️⃣ 盤中（9:30~13:00）

### 持股：依課程 5 個停損訊號監控

每隔 30-60 分鐘 quick check：

| 訊號 | 課程依據 | 觸發 |
|---|---|---|
| ① 收盤跌破關鍵低點 | 收盤確認 | 盤中接近 → 警戒 |
| ② 大黑K完整包覆前段漲幅 | K 棒型態 | 盤中收黑大實體 |
| ③ 趨勢特徵消失（低點不再走高）| 結構 | 盤中低點破前低 |
| ④ 跳空缺口當天回補失敗 | 攻擊失敗 | 開高後跌破缺口 |
| ⑤ **MA 將下彎（扣抵預警）** | 扣抵原理（早 1-2 天）| 已預警則加強監控 |

→ 出現 1 個 = **警戒**，2 個以上 = **強烈警示**，課程紅線：**最終以收盤價判斷**

### 觀察清單：動能持續性

對每檔重跑：
```bash
python scripts/zhuli/watchlist_intraday.py
```

看「**個股觀察指標**」是否持續 ✓（如「守 ma20」「bias 健康」「動能維持」）。

---

## 4️⃣ 尾盤（13:00~13:30）

### G 隔日沖進場時點（依 §G spec）

**1:20-1:25 切入**（課程明定）

前提（盤前已確認）：
- 大盤 + OTC 量增紅 K + > 5ma ✓
- 個股符合 §G spec（布林窄 / 長紅 / 量 / 月線斜率）

```bash
# 跑 G scanner 看今日是否有命中
python scripts/zhuli_scanner.py --signal overnight_swing --date $(date +%Y-%m-%d)
```

### 持股：13:25 最終決定

依課程「**收盤確認**」原則：
- 5 個停損訊號是否完整成立？
- 扣抵預警是否變紅燈（MA 已下彎）？
- 共識分數是否大幅下滑（過去 30 天 scanner active 數減少）？

→ 課程沒明說「**幾點下決定**」，但實務上等到 13:25-13:30 才能確認收盤。

---

## 5️⃣ 盤後（13:30+）

### 必做：跑 scanner 更新明日清單

```bash
# 全 scanner 跑
python scripts/zhuli_scanner.py --signal swing_breakout --date $(date +%Y-%m-%d)
python scripts/zhuli_scanner.py --signal institutional_swing --date $(date +%Y-%m-%d)
# ... 各 scanner 依需求
```

### 必做：sanity check 確認沒回退

```bash
python scripts/zhuli_scanner.py --sanity-check --all
```

預期 PASSED（0 unexpected miss）。

### 必做：更新 watchlist 清單（人工）

依今日 scanner 輸出 + 共識分數 + 課程框架，**人工挑選**明日觀察清單：
- 主力大老師「焦點股」清單（有的話）
- 跨 scanner 共識 score ≥ 2 的個股
- 持股中扣抵預警 🔴 的（明日加緊監控）

更新 `WATCHLIST` 至 `scripts/zhuli/watchlist_intraday.py`。
