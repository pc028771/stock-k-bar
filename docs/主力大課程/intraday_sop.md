> ⚠️ **CLAUDE.md 紅線**：本 SOP **不指定**具體進場價、停損價、倉位比例、盤中時機 — 這些課程沒明說的，保留給 user 自己依個股結構判斷

## 一日交易節奏（5 階段）

| 階段 | 時段 | 主任務 |
|---|---|---|
| 1. 盤前 | 8:30~9:00 | **看試撮 + 老師清單，掛單關鍵時段** |
| 2. 開盤 | 9:00~9:30 | 對照昨夜推測 vs 開盤實況 |
| 3. 盤中 | 9:30~13:00 | 持股監控（M 收高開低 警示）、觀察清單動能追蹤 |
| 4. 尾盤 | 13:00~13:30 | **「老師主題 + 結構優異」收盤前試單時機** |
| 5. 盤後 | 13:30+ | 收盤確認、跑 scanner 更新明日清單 |

## ⭐ 開盤前 8:30 試撮 SOP（黃金時段）

**Why：** 老師凌晨/清晨文章如果推主推個股（如 5/22 凌晨「ABF 主題 → 3037 欣興 + 6239 力成」），這些股 9:00 開盤常**直接鎖漲停買不到**。8:30 試撮是唯一進場機會。

**5/22 失誤案例**：6239 力成凌晨被點名，9:00 開盤直接鎖死 +9.9% 整天無法成交。

### 步驟

1. **起床第一件事（7:00-8:00）**：看老師凌晨文章（要設 watcher push 通知）
2. **8:30 試撮開始** — 對每檔老師主推查試撮
   ```python
   from clients.fubon_client import FubonClient
   FubonClient().get_realtime_snapshot(ticker)
   # 試撮回 close 有值但 open/high/low=0
   ```
3. **試撮 +5% 以上** 且**老師主推（L4-L5 或文章直接點名）**：
   - 8:50 前掛 **漲停買單** 排隊
   - 試單 1/3 部位（保守策略）
4. **9:00 開盤**：
   - 鎖死 → 看 5 檔排隊位置，沒輪到就放棄
   - 沒鎖 → 開盤後 5-10 分鐘看走勢決定加碼或撤
5. **8:30-9:00 不要在 Line 群閒聊**，這 30 分鐘決定當天能不能上車

### 試撮分級

| 試撮幅度 | 老師背書 | 動作 |
|---|---|---|
| +9% 以上 | L4-L5 主推 | 漲停掛買試單 1/3 |
| +5~9% | L4-L5 | 開盤後評估，9:05 前決定 |
| +3~5% | L4 | 觀察開盤強度 |
| < +3% | 任何 | 開盤後再看 |
| 試撮 -3% 以下 | 主推 | ⚠️ 開盤可能殺，**不掛買** |

## 1️⃣ 盤前（8:30~9:00）

### 必做：跑 watchlist 確認狀態

```bash
python scripts/zhuli/watchlist_intraday.py
```

**看什麼：**
- **共識分數**（30D 過去 multi-scanner 命中）
  - 🟢🟢 score 3+：歷史 PF > 2.10，可優先關注
  - 🟡 score 1：baseline 訊號
  - ⚪ score 0：無 multi-scanner 確認，需單獨評估
- **扣抵預判燈板**
  - 🔴 MA 將下彎 = **早期出場警示**（比課程「趨勢特徵消失」早 1-2 天）
  - 🟡 MA 臨界：1-2 天內可能轉折
- **個股觀察指標** ✓/✗ 課程框架檢核

### 必做：大盤面確認（課程 Ch6-1）

依 §G 隔日沖 spec：
- 加權指數：量增紅K + 收盤 > 5ma
- OTC 指數：量增紅K + 收盤 > 5ma
- **兩者皆符** = 大盤友善，當沖/隔日沖可考慮
- **任一不符** = 大盤偏空，僅保留中長波段策略

## 2️⃣ 開盤（9:00~9:30）

### 持股檢核：M 收高開低 出場警示

對每檔持股檢查（**用昨日為基準**）：
- 昨日是否「收最高 + 出量」？
- 今日開盤是否 ≤ 昨收？

→ 若 **M bearish_exit + 漲停板平盤警示同時成立** = 主力試空雙重警示
- 課程紅線：**收盤確認**，盤中震盪非依據
- 警示後等收盤再決定

### 觀察清單：開盤跳空判讀

依課程 Ch2-3：
- **跳空向上**（gap_up）：今 low > 昨 high → 多方缺口，缺口低為支撐
- **跳空向下**（gap_down）：今 high < 昨 low → 空方缺口，缺口高為壓力

## 3️⃣ 盤中（9:30~13:00）

### 持股：依課程 5 個停損訊號監控

每隔 30-60 分鐘 quick check：

| 訊號 | 課程依據 | 觸發 |
|---|---|---|
| ① 收盤跌破關鍵低點 | 收盤確認 | 盤中接近 → 警戒 |
| ② 大黑K完整包覆前段漲幅 | K 棒型態 | 盤中收黑大實體 |
| ③ 趨勢特徵消失（低點不再走高）| 結構 | 盤中低點破前低 |
| ④ 跳空缺口當天回補失敗 | 攻擊失敗 | 開高後跌破缺口 |
| ⑤ **MA 將下彎（扣抵預警）** | 扣抵原理（早 1-2 天）| 已預警則加強監控 |

→ 出現 1 個 = **警戒**，2 個以上 = **強烈警示**，課程紅線：**最終以收盤價判斷**

### 觀察清單：動能持續性

對每檔重跑：
```bash
python scripts/zhuli/watchlist_intraday.py
```

看「**個股觀察指標**」是否持續 ✓（如「守 ma20」「bias 健康」「動能維持」）。

## 4️⃣ 尾盤（13:00~13:30）

### ⭐ 「老師主題 + 結構優異」= 收盤前試單

**Why：** 老師盤中/白天明確點名 + 個股收盤前已展現「結構優異」（量縮紅K、量增突破、上影線少、距高點近），13:25 收盤前試單 1/3 是最佳時機。

**5/20 群創教學影片案例**：5/20 老師教學影片講 N字上攻（群創 3481），5/20 群創收 37 量比 1.10（量縮整理末端）→ **如果 5/20 收盤前試單 @37**，5/21 漲停 +9.9% 賺到。

### 判斷流程（13:00-13:25）

1. **老師當日訊號**（白天 Line 群、培訓、文章有沒有明確點名）
2. **個股收盤前結構**（13:00 看當下 OHLC）
   - 量比 < 1.5（量縮配合）
   - 收盤接近最高（body > 0、上影 < 2%）
   - 距 10 日高點 < 5%
   - 站穩 MA5
3. **保守策略過濾**
   - 啟動第 1-2 根 ✓
   - 第 3 根之後 → 放棄
4. **13:25 決定**：
   - 都通過 → 試單 1/3 部位
   - 任一不通過 → 不進

### 收盤前試單 vs 開盤前試撮對比

| 情境 | 收盤前試單 | 開盤前試撮 |
|---|---|---|
| 何時用 | 老師白天就講了 | 老師凌晨文章新點名 |
| 風險 | 隔日開盤可能跳空高 | 開盤可能直接鎖死 |
| 成本 | 收盤價（明確）| 漲停掛買排隊（可能買不到）|
| 適合度 | 結構已成形時 | 主題突發、剛點名時 |

兩個都是「跟隨老師訊號 + 個股結構」的進場時機，差別只在訊號發布時間。

### G 隔日沖進場時點（依 §G spec）

**1:20-1:25 切入**（課程明定）

前提（盤前已確認）：
- 大盤 + OTC 量增紅 K + > 5ma ✓
- 個股符合 §G spec（布林窄 / 長紅 / 量 / 月線斜率）

```bash
# 跑 G scanner 看今日是否有命中
python scripts/zhuli_scanner.py --signal overnight_swing --date $(date +%Y-%m-%d)
```

### 持股：13:25 最終決定

依課程「**收盤確認**」原則：
- 5 個停損訊號是否完整成立？
- 扣抵預警是否變紅燈（MA 已下彎）？
- 共識分數是否大幅下滑（過去 30 天 scanner active 數減少）？

→ 課程沒明說「**幾點下決定**」，但實務上等到 13:25-13:30 才能確認收盤。

- 跨 scanner 共識 score ≥ 2 的個股
- 持股中扣抵預警 🔴 的（明日加緊監控）
