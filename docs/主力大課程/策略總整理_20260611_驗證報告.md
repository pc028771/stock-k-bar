## ⚠️ 有問題的部分（按嚴重度排序）

### 🔴 嚴重：紀律疏漏（會影響實際交易判斷）

#### 1. 跳空紅線混淆兩個情境
- **總整理寫**：紅線 #1 跳空 ≥ +5% 不觸發
- **實際**：老師紀律有**兩條獨立**規則
  - **漲停隔日跳空 ≥ +3%** → 禁推（`feedback_entry_timing_rules`、4526 -$12,860 教訓）
  - **一般跳空 ≥ +5%** → code 化紅線
- **影響**：把 +3% 鎖漲停案例當 +5% 處理、會放行不該推的進場
- **修正**：加紅線 #4「鎖漲停隔日跳空 ≥ +3% skip」

#### 2. 短線進場三鐵漏第三鐵
- **總整理 code 化紅線**：跳空 / 距 MA10 / 前 10 分
- **實際三鐵**（`feedback_short_swing_entry_discipline`、5/19 實戰課）：
  - ① 雙錨停損 ✅
  - ② 前 10 分鐘不切入 ✅
  - ③ **前 5 分鐘漲幅 > 5% = skip（一票否決、整檔今天不碰）** ❌ **缺**
- **修正**：加紅線 #5「前 5 分鐘漲幅 > 5% → 整檔今天 skip」

#### 3. 「尾盤才進場」鐵則沒寫成跨 trigger 規則
- **總整理寫**：Closing trigger 13:05-13:25 進場
- **實際紀律**（`feedback_close_session_only_entry`、user 6/5 痛定思痛、8046 -$62k 教訓）：
  - **09:00-12:59 純觀察、不下任何進場單**（含首攻 Ch5-3、續攻 T1、反彈 T2）
  - 13:00-13:25 才是唯一進場窗口
  - 例外：老師明示「現在切」+ broker tier 1 confirm
- **影響**：總整理列了 Ch5-3 / T1 / T2 是「進場 trigger」、但實際上盤中不該執行
- **修正**：在「紀律」章節加跨 trigger 鐵則

#### 4. 警示族群完全沒提
- **總整理**：零提及
- **實際**：散熱 / 連接器 / PCB 自 5/17 起連續 4 週警示（`feedback_warning_sectors_exclusion`、`feedback_warning_sector_not_complete_skip`）
- **規則**：不完全排除、加 ⚠️ 標籤、50% sizing
- **修正**：加「警示族群」獨立章節

#### 5. 加碼 +10% 通則的標籤錯誤
- **總整理寫**：「未 code 化」
- **實際**：這是**紀律規則**、不是 code 能不能做的問題（user 已決定靠人工把關 Stage 2、避免 AI 自動加碼）
- **修正**：標籤改「人工紀律（user 6/5 鎖定）」

### 🟡 中度：課程依據混淆（沒寫錯但會誤導）

#### 6. Trigger 代號「Ch5-3 / T1 / T2 / TC / Closing」是內部命名、非課程術語
- 課程講稿用「首攻 / 續攻 / 反彈 / 破底 / 尾盤」中文表達
- 代號是 backtest / monitor 內部用、應標註「（內部代號）」

#### 7. Closing「3-4/5 = Win 82% / 5/5 = Win 40%」勝率數字
- **無課程出處**、是 user 自家 backtest 結論（`feedback_exit_rules_v3`）
- 應標註「（backtest 結果、非課程）」

#### 8. 均線全順向 3 項距離門檻（12% / 40% / 25%）
- 課程只教「均線全順向」概念
- 12% / 40% / 25% 是反向工程的濾網、是實作補強
- 應標註「（實作參數、非課程明授）」

#### 9. 雙軸籌碼相對強勢缺實作細節
- **總整理術語表寫**：「大盤弱時找相對強」
- **實際**（`feedback_dual_axis_relative_strength`）：
  - 觸發前提：TAIEX ≤ -1%
  - 條件 AND：外資 5d ≥ +3,000 張、投信 5d ≥ -500 張、站均線
  - 行動：只標記、不過濾
- 應補完整實作描述
