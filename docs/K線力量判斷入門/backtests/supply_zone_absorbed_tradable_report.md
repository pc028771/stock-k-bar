# supply_zone_absorbed 可交易版本驗證（Task 14）

資料庫：`/Users/howard/.four_seasons/data.sqlite`

樣本：2025-01-02 至 2026-05-08

---

## 1. 問題背景

`supply_zone_absorbed`（賣壓化解）在 `supply_zone_spec_report.md` 中展示出極強的預測力（10日 +7.52%、勝率 64.7%），但該欄位使用了**未來 5 日的收盤資料（look-ahead）**，無法作為即時交易訊號。

Task 14 目標：將其改為可交易版本，觀察延遲 5 個交易日確認後再進場（t+6 open）的實際績效。

---

## 2. 設計說明

### 原版（前瞻標註，僅供回測比較）

| 條件 | 定義 |
| --- | --- |
| 突破 60 日高點 | `close > prior_high_60` |
| 化解確認（look-ahead） | 未來 1~5 日最小收盤 ≥ `prior_high_60 × 0.98` |
| 進場時間 | t+1 open（前瞻標註，非可交易） |

### 可交易版本（Task 14）

| 條件 | 定義 |
| --- | --- |
| 突破 60 日高點 | `close > prior_high_60`（第 0 日，訊號日） |
| t+5 確認守住 | 等待 t+1~t+5 最小收盤 ≥ `prior_high_60 × 0.98`（即 `supply_zone_absorbed` = True） |
| 進場時間 | t+6 開盤（確認守住後隔日開盤） |
| 出場時間 | t+6 進場後第 5/10/20 個交易日收盤 |

**核心差異：** 原版在 t+0 當天就用未來資訊確認，可交易版等到 t+5 收盤才能確認，t+6 才能進場。

## 4. 結論

### 4.1 可交易版績效大幅衰退

原版的強績效（10日 +5.70%、勝率 60%）主要來自**前瞻確認的資訊優勢**：它已知道 t+5 日前沒有跌回，等於提前剔除了失敗突破。一旦改為可交易版（t+6 進場），這個資訊優勢消失，績效回歸接近市場基準（10日 +2.67%，勝率接近 50%）。

### 4.2 仍有邊際但不顯著

可交易版的 20日報酬仍為正（+6.0%），`range` regime 下略好（+6.87%、勝率 51%），但 10日勝率不足 50%，統計上不顯著。

### 4.3 課程語意對照

課程（strategy-indicators.md §4）說「賣壓化解要觀察股價是否能有效穿越過往套牢區，且**穿越後不快速跌回**」。可交易版正確地等待「不快速跌回」確認（t+5 站穩），但此確認本身已反映在 t+1~t+5 的價格上，導致 t+6 進場時溢價過高，邊際效益大幅降低。

### 4.4 主要限制

- 樣本期間（2025-2026）大部分為 bull/range 環境，bear 樣本偏少。
- t+6 進場點的確切「溢價程度」未分析；若 t+5 收盤相對突破點的漲幅已超過 5%，進場風險更高。
- 課程未說明「確認站穩後的具體進場時機」，此版本以 t+6 open 為代理，符合課程「收盤確認後隔日開盤進場」的基本框架。

---

## 5. 建議

- `supply_zone_absorbed` 原版作為**回測分析的標註標籤**使用，不直接作為交易訊號。
- 可交易版可以與 `breakout_attack + overhead_supply_layer ≤ 1` 組合使用，選出突破後站穩、上方賣壓少的候選，但不應視為獨立進場依據。
- 後續若能取得 FinMind 分K資料，可考慮改用「t+5 日的盤中走勢確認」來縮短延遲。
