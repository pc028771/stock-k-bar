## 核心：K 線圖上沒有「支撐」

> 「K 線圖上**只有壓力、沒有支撐**……壓力是確實的存在，但是支撐卻是一種一廂情願的看法。」

例外：
- 五日線、季線等均線**都不是**支撐
- **只有十年線**肯定有國安基金進場

老手經驗法則：
> 「壓力是用來被突破的、**支撐是用來被跌破的**」
> 「趨勢成形的時候，什麼壓力都壓不住、什麼支撐都撐不下去。」

## 缺口支撐的真實意義

> 「缺口支撐型態判斷的關鍵並不是原本的趨勢，而是這個缺口出現的背景**「是否是因為利多消息。」**」

兩種狀況：
1. **空方趨勢中的跳空（利多帶上來）** → 不可作為支撐
2. **多方趨勢創新高的攻擊跳空** → 不可被回補

## 空方下利多跳空的陷阱

範例：南帝(2108) 111-03-10
- EPS 14.92 + 配 7 元現金股利
- 兩個向上跳空 → 「缺口支撐感」
- 投資人停損設在 71 元（缺口下緣，比進場價 81.3 低 10 元）
- 結論：停損距離太大，本來想買低就不會輕易停損
- 兩個月後：被迫除息 + 股價繼續走低 → 接近腰斬

> 「會想買在低檔，就是因為覺得股價偏低有進場價值，當股價更低就會反射性地認為更有價值了，又怎會輕易停損？」

## 攻擊跳空的「不可回補」

範例：凌網(5212) 109-04-27
- 「跳空突破」（採用向上跳空突破前高）
- **不能再回頭**的型態
- 「主力本來可以慢慢拉出突破前高的長紅，散戶不會在這裡才追逐，所以不用這麼明顯的用跳空來突破前高」
- 既然用跳空突破 → **沒道理又回頭讓散戶有低可買**

## 真正的「缺口支撐」判定

> 「**最簡單的判斷就是明顯的不會再碰到，才有支撐意義**。」

> 「**正確的邏輯是確認不再回到缺口以下的價位才是真正的缺口支撐**，可是對於想要當作買進理由的人來說，確認要等股價都上去了，就感覺好像不實用。」

## 對系統的啟示

### 1. 完全禁止「缺口支撐買進」邏輯

加入禁止項：
- ❌ 因為「股價在跳空缺口上」就視為支撐買進
- ❌ 利多消息驅動的跳空 = 支撐買進依據
- ❌ 任何「均線支撐買進」（五日、月線、季線）

### 2. 攻擊跳空被回補 = 確認失敗

已實作 `gap_attack_filled` exit condition。

### 3. 「跳空突破」是負分

對應行進ing 跳空篇(三)：「跳空突破不理想」。

```python
def is_gap_breakout_weak(df):
    """
    跳空突破（一日內完成）= 弱信號
    """
    is_gap_up = df["open"] > df["prev_high"]
    is_new_high = df["close"] > df["prior_high_60"]
    # 突破與跳空同日 = 一日完成 → 弱
    return is_gap_up & is_new_high
```

→ scanner ranking 應扣分（vs 突破跳空：先突破後跳空為強）

### 4. 「利多 + 跳空」=「不可作為支撐」標記

```python
def is_news_driven_gap_up(df):
    """
    利多帶上來的跳空，不能視為支撐
    
    Note: 需要新聞資料整合或用「事件型大量跳空」推測
    """
    is_gap_up = df["open"] > df["prev_high"]
    is_oversized_gap = (df["open"] / df["prev_close"] - 1) > 0.05  # >5% gap
    # 如有量資料：是否為 N-day volume max
    return is_gap_up & is_oversized_gap
```

### 5. 「真正不會再碰到」= 攻擊延續確認

對應日出攻擊：突破後連續日出（high > prev_high AND low > prev_low）= 確認沒有回頭意圖。
