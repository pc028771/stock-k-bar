## 「騙線」是錯誤稱呼，正確是「陷阱」

> 「騙線型態的正確說法應該稱之為**陷阱**，也就是可能會發生誤判的型態。」

林家洋立場：
- 「割韭菜」、「騙線」都是「**錯的不是我**」的推託之詞
- 並沒有任何資金真的花錢去騙人
- 是投資人理解不夠深入

## 區分：什麼**不是**騙線

> 「如果單純是發生了某些現象卻又消失，這很正常並不算是騙線，例如突破頸線之後創新高，很快股價又跌回頸線之下，這**不是騙線**，只是市場資金並沒有人打算要拉高股價而已。」

**真正的騙線（陷阱）**：突破頸線後上方依然有套牢壓力時，才會發生誤判。

## 三大類陷阱

### 一、上有壓力的突破

最常見的一種，符合頸線定義但**上方還有明顯套牢區**。

範例：中石化(1314) 107-05-18
- 股價 < 15 元
- 跳空紅K真的是頸線突破
- 但上方還有過往高檔套牢
- 因低價而被當作買進機會 → 誤判
- 半年後驗證

**鐵則**：低價股的「突破」最容易被當作沒看到上方賣壓。

### 二、看似攻擊的回頭

範例：合勤控(3704) 108-04-26
- 跳空缺口因利多訊息出現（被警示後公告月 EPS）
- 四天後缺口被回補
- 「對於短線交易者來說，利多是沒有用的」
- **主力拉抬過的角度，利多正好是出貨時機**

### 三、假性跌破（限大盤）

> 「個股跌破頸線之後頭部成形，套牢賣壓就是真實的存在了。**而大盤往往要在多數股票都已經空頭、資金也越來越缺乏拉抬個股的意願**，才會原本是假性跌破，卻後來又真正的跌破頸線了。」

→ 與「行進ing 黑K篇(四)」一致：個股無假性跌破。

**識別假性跌破**：對多空波動原理有正確認識（連續急跌後反彈站回頸線）。

範例：111-03-07 大盤 K 線圖跌 557 點，沒有騙線機會（不要思考太多）。

## 核心立場

> 「人心會有一種明明是錯誤，卻解讀成自己被騙了的安慰心理，這一點需要盡快導正。」

## 對系統的啟示

### 1. 入場 filter：「上有壓力的突破」必須過濾

```python
def is_clean_breakout(df):
    """突破上方無套牢"""
    is_breakout = df["close"] > df["prior_high_60"]
    is_clean = df["overhead_supply_layer"].fillna(0) <= 0
    return is_breakout & is_clean
```

→ 這是 scanner 排除騙線的最有效 filter。

### 4. 個股不適用假性跌破已落實

`false_breakdown_reclaim` 只用於大盤層次（行進ing 已強調）。

### 5. 別過度設計「真假突破」判斷

> 「不要思考太多假性與不假性的問題，因為**考慮了太多例外，會把正常應該要判斷的時機錯過**。」

→ 系統設計上，跌破訊號就觸發出場，不要試圖判斷「這次是真破還是假破」。

### 6. 低價股的陷阱

> 「低價股往往會有這樣的假象，因為價格低，人們總愛只看自己想看的那一段。」

→ scanner 可加 column `is_low_price`（close < 20）並對突破類型加更嚴格 filter
