## 破底型態定義

對應「再突破」的空方版本：
- 多方再突破 = 突破不只一次
- 空方破底 = 跌破不只一次低點

## 結構：層層套牢

> 「破底型態可以說是中樞型態在空方狀態中的進一步把區間加大，因為低點又被跌破。」
> 「結構上，破底型態需要認識的本質，是在 K 線圖上就是又**多增加了新的一層套牢區段**。」

每次破底 = 多一層套牢區。

## 破底原因（必須知道）

兩大類：

### A. 環境因素（共同破底）
- 力量缺乏導致連續弱勢
- 跌勢過程量小 → **沒形成大障礙**
- 風險相對較小

### B. 個股個別問題
- 不會無故連續創新低
- 共通點：基本面或營運獲利問題
- **碰觸這樣的股票只是在給自己找麻煩**

## 範例

### 康控(4943) 110-11-26
- 從 624 跌到 34（後續還腰斬到 15 以下）
- 原因：EPS 由盈轉虧
- **不能因「比較效應」感覺低就買**

### 富邦媒(8454) 111-05-17
- 題材股炒作 → 還原真正價值
- 上方多層套牢
- 每次反彈都不越過上方水平線

### 大立光(3008) 111-05-17
- 本益比修正式下跌
- 需要原因消失才可能改變

## 改變破底型態的唯一方法

> 「股價要如何改變這種破底型態？只有一個辦法，就是當**本益比修正式下跌的條件不存在了**，才有可能改變這種波動狀態，不過賣壓仍是阻礙。」

技術面：等**空方趨勢結束**。

## 「人多的地方不要去」是次要

> 「更重要的是對於投資而言，『**危險的地方**』不要去，就算人很少也不能去。」

## 鐵則

> 「離現在最近一個壓力越過、或者空方趨勢結束之前，**都不能有摸底的想法**。」

## 對系統的啟示

### 1. 「過去 N 天有破底紀錄」= 直接排除候選

```python
def is_in_breakdown_pattern(df, lookback=60):
    """
    破底型態 = 過去 N 天有 >= 2 次破前低 + MA60 下彎
    """
    g = df.groupby("ticker")
    is_new_low = df["low"] < g["prior_low_20"]
    breakdown_count = is_new_low.rolling(lookback).sum()
    is_ma60_down = df["ma60_slope_5d"] < 0
    return (breakdown_count >= 2) & is_ma60_down
```

→ scanner 應**硬性排除**破底型態中的個股。

### 2. 「人去樓空」+「破底型態」一致

對應行進ing 黑K篇(五) 已有的觀念，這裡更明確：層層套牢加上每次反彈不越過上方水平線。

### 3. 「本益比修正式下跌」概念

需要基本面整合：
- 過去 EPS 走勢
- 股價 / EPS 比例變化
- 「本益比過高 + 開始連續創新低」= 修正式下跌警示

### 5. 等空方趨勢結束的明確條件

- MA60 由下彎轉上揚（季線扣抵預判 + 確認）
- 同時越過最近壓力位

→ 對應現有的「entry/trend_reversal」STUB 的精確要件。

### 6. 「比較效應」陷阱

> 「不能因『比較效應』感覺當初都從高價跌到現在這麼深了，應該會有機會就進場。」

→ 加入禁止 pattern：「**從高價跌深 = 機會**」邏輯。

### 7. 「曾經操作過」的個股偏見

> 「人們往往會對於自己操作過的股票、曾經賣過高價的，留下美好的印象；當初曾經在高檔區域賣過高價，現在就會有一種低檔機會又來了的錯覺。」

→ 系統反而占優勢：**沒有歷史持倉偏見**。
