## 核心立場：楔形是最不重要的型態

> 「楔形型態在所有的型態之中是**最不重要**的一種，原因在於與其他的技術分析理論有大面積的重疊，例如短期趨勢，或者多空波動原理。」

## 楔形定義

「行進整理的區間，**有角度的往上或者往下**」（不是橫向 = 不是箱型）

繪製：依波段狀態的高點與低點個別連線（**不是精確平行**）

→ 既然是用波動原理連線，**為什麼不直接用多空波動原理？**

## 上升楔形 = 加碼點輔助（唯一有用處）

> 「對於上升楔型來說唯一的優勢可能就是加碼點的輔助判定。」

理由：
- 上升楔型過程中股價持續創新高 = 主力用力拉抬中
- 主力沒有出貨疑慮
- 在下緣附近加碼，停損點明確

範例：燦坤(2430)、聯華(1229)

**缺點**：果斷加碼遇長黑跌破下緣，會比單純持有損失更大。

## 下降楔形 = 完全雞肋

> 「下降楔行有時候會帶給技術分析的使用者有多此一舉的感覺，因為型態的角度看來似乎有意義，實際上操作時根本用不上。」

問題：
- 已經成為下降楔形時，**頸線早已跌破**
- 季線早已下彎
- 突破下降楔形上緣作買點 = 忽略所有套牢賣壓

→ 等於「不要摸底」的提醒，但用多空波動原理就看得出來

## 警示：不是 K 線有支撐

附錄案例：聯強(2347) 111-06-09
- 公告五月營收歷年最佳
- **直接跳空向下**
- 不是公司有問題
- **股價沒有拉抬的力量存在其中時，看不到支撐是常見的**

→ 即使有上升楔形 + 利多，沒有拉抬力量也會跌下來

## 加碼為什麼不值得推崇

「**加碼並不是一種很值得推崇的作為**」——但技術分析角度，只有遇到上升楔形有加碼機會。

→ 對應入門系統「不能攤平」「不能來回操作」的禁忌。

## 對系統的啟示

### 1. 系統暫時不實作楔形偵測

依課程立場：
- 上升楔形：只用於加碼判斷，與目前系統定位（純多方進場）不對齊
- 下降楔形：完全雞肋

→ 不需要 `is_rising_wedge` / `is_falling_wedge` 偵測

### 2. 但要區分「箱型」vs「楔形」

`is_pattern_breakout` 的條件中應該排除楔形（高低點漸高/漸低的整理）：

```python
def is_pattern_breakout_strict(df):
    # 條件：過去 60 日不能有「明顯漸高/漸低」的趨勢
    # 否則屬於楔形，不是箱型
    is_box = (rolling_max_slope < threshold) & (rolling_min_slope < threshold)
    is_breakout = df["close"] > df["prior_high_60"]
    return is_box & is_breakout
```

### 3. 「股價沒有拉抬力量就沒有支撐」鐵則

範例聯強——好基本面 + 上升楔形 + 利多月營收 → 仍跳空向下

→ 對應「**沒有支撐**」鐵則：系統不能用任何「支撐」概念當依據（已遵循）

### 4. 短期趨勢線是楔形的替代

> 「單純就用短期趨勢來看待，有什麼不足之處嗎？當然沒有」

→ 系統可實作「短期趨勢線跌破」（多空波動的前低跌破）取代任何楔形相關偵測
