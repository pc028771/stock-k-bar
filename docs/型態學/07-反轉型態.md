## 反轉型態 = 多空轉折組合（力竭）

> 「反轉型態更完整的說法就是『**多空轉折組合**』。」
> 「這是一項基於**力竭原理**的組合 K 線研判，用途當然只是**原有持有部位的出場使用**。」

## 一眼辨別的兩種優先

### 空方反轉（必會）
- **黑K吞噬**
- **高檔長黑**

「所謂的高檔，已經結合了過去一段時間股價是在攻擊狀態的，那麼力竭代表了主力準備開始出貨的跡象。」

### 多方反轉（限低檔背景）
- **母子晨星**（最常見）
- **紅K吞噬**（實務上很少見）

「多方反轉之前的背景，通常都是環境氣氛不佳，股價連續且漫長的崩跌，人心最脆弱的時期出現。」

## 反轉型態 ≠ 立即價格反轉

> 「反轉型態是名稱，不是一定要馬上價格反轉，因為多方的力量要轉變為空方的過程也不會瞬間改變，但是力量的態勢，已經大不相同了。」

範例：智晶(5245) 110-12-14
- 空頭吞噬出現後**還有再創過一次新高**
- 但高檔的黑K已知多方力量心態完全改變
- → 反轉型態的真正意義 = 「力量態勢的改變」，不是價格立即下跌

## 多方反轉**不是買點**（重要）

> 「股價如果遇到了空頭趨勢下跌，只要持續得夠久加上環境因素，往往會**超跌**。這是我們不應該用多方轉折組合來當作買點的原因，可以看出空方的力竭已經是很大的幫助。」

範例：欣銓(3264) 111-04-13
- 連續兩次出現母子晨星
- 事後股價都還有更低
- 不是型態判斷錯誤，是**市場氣氛沒有多方力量**
- 加上**沒有賣壓中空區段**

範例：南帝(2108) 111-03-09 「失敗的島狀反轉」
- 同樣是「缺乏賣壓中空狀態」造成失敗

## 多方反轉的「再加邏輯」

> 「多方反轉型態需要再加上一個邏輯，就是紅K與黑K組成的要素不同，紅K需要有買盤的力量進駐，但是下跌往往沒人買就可以再跌下去。」

→ 多方反轉的可靠度本質上低於空方反轉。

## 多方反轉的必要前置

1. 連續且漫長的崩跌（環境氣氛肅殺）
2. **賣壓中空區段**已存在（不在套牢區）
3. 紅K要有買盤力量（非真空跌）

## 對系統的啟示

### 1. 反轉型態類別分類

| 用途 | 課程立場 | 系統實作 |
|---|---|---|
| 空方反轉（黑K吞噬、高檔長黑、空頭吞噬、夜星棄嬰、跳空反轉、雙鴉躍空、暗夜雙星）| **必會** | ✅ 已在 T32 全部實作 |
| 多方反轉（母子晨星、紅K吞噬、多頭吞噬）| **不是買點**，只是空單回補/空方力竭 | trend_reversal 應為 STUB（無條件 False）或極嚴格 gate |

### 2. `trend_reversal.py` STUB 維持為 STUB 是對的

不應該實作多方反轉作為入場訊號——課程明確不建議。

如果未來要做：
- 必須有 swing low 連續一段時間（賣壓中空）
- 必須在 MA60 已下彎但即將上揚的扣抵階段
- 並且只用於價值/殖利率投資（不是短線）

### 3. 反轉型態「不立即反轉」要納入系統設計

目前 simulator 一觸發 reversal_k 就出場——這是對的，因為**力量態勢改變**是出場依據，不需要股價立即下跌驗證。

### 4. 失敗的反轉型態 = 缺賣壓中空

```python
def is_reversal_invalid(df):
    """
    多方反轉訊號若無賣壓中空，可能是失敗的反轉
    """
    has_supply_vacuum = df["supply_vacuum_zone"] == 1
    in_overhead = df["overhead_supply_layer"] >= 1
    return ~has_supply_vacuum | in_overhead
```

→ 即使未來實作多方反轉作為買點，這個 invalid gate 是必要的。

### 5. 「型態 + 力量」雙重判斷

> 「攻擊型態是最有效也容易誤解的，但是都是基於資金力量變化的意義。」

→ 入場訊號不應只看「型態」，要結合「力量」（量、紅K真實性、攻擊延續度）
