## 缺口壓力的本質

> 「向下跳空的缺口表示這個價位區間**沒有任何人成交**……這個價位卻沒有任何買單願意承接股價，於是就形成了缺口壓力的明顯壓力狀態。」

雖然不是實質套牢，但是**型態上的壓力**。

## 鐵則：壓力是確實的存在

> 「壓力是確實的存在，特別是套牢賣壓的存在，不會因為時間久了就改變。」

即使停損賣出 → 也有人接手 → 高檔仍有套牢，只是量大小差別。

## 兩種趨勢下的缺口壓力

### 多方趨勢下的缺口壓力（嚴重）

範例：潤泰全(2915) 111-05-09
- 創新高後公告 EPS 22 元但只配 5 元
- 兩根跳空跌停
- 第三根帶量長黑
- 大量原因：投資人「跌三成 + 公司沒問題」搶反彈
- → **缺乏缺口壓力型態觀念**的買進

### 空方趨勢下的缺口壓力（較輕但仍存在）

範例：網家(8044) 111-05-04
- 第一個向下跳空 + 頸線跌破 = 頭部型態確立
- 第二次跌破前低又跳空 → 缺口壓力相對較小

**規律**：越後來出現的缺口，影響力越低。

## 反彈站不上 vs 站得上的差異

範例對比：
- 潤泰全：跌三成後反彈無力（沒有賣壓中空）
- 廣明(6188) 109-01-15：利空連續跳空 → 反彈漲停（**有賣壓中空段**）

> 「缺口壓力型態不單單只是考慮向下跳空的缺口段而已，就算是反彈也需要評估真正的套牢結構位置何在，才能辨識有沒有賣壓中空段。」

## 多缺口情境的判斷規則

> 「**只要離現在最近的一個缺口壓力還沒有越過之前，就都不宜對股價樂觀。**」

→ 大盤層次：97 年範例——最近的破底缺口被回補前都不宜樂觀。

## 對系統的啟示

### 1. 新增 feature: `recent_gap_down_lower`

```python
def recent_gap_down_resistance(df):
    """
    過去 N 天最近的向下跳空缺口下緣
    
    用途：價格還沒越過此位置時，視為仍有缺口壓力
    """
    g = df.groupby("ticker")
    is_gap_down = df["open"] < g["low"].shift(1)
    # 該缺口下緣 = 跳空當日 high (或 prev_low)
    gap_down_upper = df["prev_low"].where(is_gap_down)
    return gap_down_upper.groupby(df["ticker"]).ffill()
```

→ 此為**不可入場的硬性過濾**：scanner 應排除上方有未被回補向下缺口的個股。

### 2. 增強 `overhead_supply_layer`

目前的 layer count 只看 swing-high peaks。應該加上「**未回補的向下跳空缺口下緣**」作為額外壓力來源。

### 3. 反彈評估邏輯

當前系統的反彈判斷不足。應補：

```python
def is_bounce_into_gap_resistance(df):
    """
    反彈中遇缺口壓力 → 反彈無力
    """
    # 條件 1: 過去有向下跳空缺口未回補
    # 條件 2: 今日反彈接近該缺口下緣
    # 條件 3: 該缺口下方無明顯賣壓中空段
```

→ 此狀態下的「破前高」應該視為**騙線突破**，不是真攻擊。

### 5. 利空 + 連續跳空 = 賣壓中空形成

對應入門概念：急跌段形成賣壓中空。應加 column：
- `is_in_gap_vacuum`: 當前位置處於連續跳空下跌的中間段
- `has_supply_vacuum_below_overhead`: 上方有缺口壓力但下方有賣壓中空

→ 賣壓中空段內反彈可能性大，缺口壓力到了卻會止漲。

### 6. 跳空缺口下緣 vs 套牢區下緣

兩者都是壓力，但語意不同：
- 套牢區 = 有人真實成交 → 必須化解
- 跳空缺口 = 沒人成交 → 沒人「需要」解套但也無買盤

→ 統一視為「上方壓力區」進入 ranking 扣分即可，不必細分。
