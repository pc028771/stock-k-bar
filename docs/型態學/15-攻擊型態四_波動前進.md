## 波動前進型態 = 最容易誤判的攻擊型態

> 「在所有的攻擊型態中，**最讓人失去耐心，也最容易誤判**，甚至是看不出來像是攻擊的，就是這種『緩慢波動前進』的型態。」

## 與其他型態的關係

> 「『波動前進型態』中的波動如果區間放大，就變成了『**上升楔型型態**』。」
> 把上升楔型壓縮到看不出區間（K 線黏在一起）= 波動前進型態。

→ 三者只是「波動幅度」程度不同：
- 黏緊（看不出來）= 波動前進
- 略可見 = 推升攻擊
- 幅度大 = 上升楔型

## 四種攻擊型態的強度排序

| 強度 | 攻擊型態 | 辨識依據 |
|---|---|---|
| 最強 | 日出攻擊 | 每天 high + low 都往上 |
| 次強 | 跳空攻擊 | 突破紅K隔日跳空且不回補 |
| 中 | 推升攻擊（低點推升）| 連續 K 線**低點**逐步往上 |
| **最弱（黏著）** | 波動前進 | 連續 K 線**高點**逐步往上 + 多數 K 線與前一天重疊 |

## 辨識規則（重要）

> 「依照程度來分類，價格最黏著的就是波動前進型態，**辨別採用 K 線的高點是否持續創新高**判斷；稍微看得出來橫向整理的，就是看低點的推升，辨識採用 K 線的低點是否有著推升意義；如果幅度大一點就會是楔型型態。」

→ 三種 swing 變化的程式判別：
- 波動前進：`is_higher_high_streak`（高點推升，幅度小）
- 推升攻擊：`is_higher_low_streak`（低點推升）
- 楔形：上下緣連線都向上（範圍大）

## 範例

### 廣積(8050) 111-08-30
- 突破後波動前進
- 每根 K 線幅度小，看不出強烈攻擊
- 經歷一段時間後才發現高點逐步上移

### 遠東新(1402) 110-04-26
- 所有 K 線幾乎黏在一起
- 多數 K 線與前一天重疊
- 但連續 K 線**高點**依然體會到些微攻擊

## 短期上升趨勢線輔助

> 「型態的判斷是一段時間的狀態，**可用趨勢線作為輔助確認**是否往上波動。」

## 「股性」的概念與放棄判斷

範例：義隆電(2458) 110-03-26
- 主力刻意製造看不出來的方式
- 創新高 → 震盪 → 看似空方 → 又多方
- 採用型態學進場會「得不償失」
- **這種類型就不屬於可以運用型態學來判斷的走勢**

→ 系統設計上需要**識別並排除**「股性混亂」的個股。

## 型態的真正功用

> 「型態多數並不是作為買點或者賣點使用，而是**確定股價目前的狀態而已**。」

## 對系統的啟示

### 1. 波動前進辨識：高點推升

新 feature：

```python
def higher_high_streak(df, lookback=10):
    """連續高點推升的天數"""
    is_higher_high = df["high"] > df["prev_high"]
    return is_higher_high.groupby(df["ticker"]).rolling(lookback).sum()
```

### 3. 「股性混亂」的識別與排除

```python
def is_chaotic_pattern(df, lookback=60):
    """
    股性混亂 = 高頻次的趨勢翻轉 + 創新高與破底交替
    
    特徵：
    - 過去 60 天有 N 次以上 high > prior_high_20
    - 同時有 N 次以上 low < prior_low_20
    - MA60 方向變化 > 2 次
    """
    g = df.groupby("ticker")
    new_high_count = (df["high"] > g["prior_high_20"]).rolling(lookback).sum()
    new_low_count = (df["low"] < g["prior_low_20"]).rolling(lookback).sum()
    return (new_high_count >= 5) & (new_low_count >= 5)
```

→ 識別後 scanner 排除這類個股。

### 5. 「持有判斷」的優先級

對應行進ing 反覆強調：型態學的真正功用是**確定目前狀態**，不是買賣點。

→ Scanner 排序加 ranking：
