## 跳空攻擊定義（精確）

> 在出現了突破的紅K，股價創下新高之後，**隔天的開盤就跳空開出，且直到收盤都沒有回補缺口**。

關鍵：**直到收盤都沒有回補缺口**——這是定義中重要的一句。

## 失敗情境

開盤跳空但盤中回補缺口 → **跳空攻擊失敗**
- 但不代表整個攻擊走勢失敗
- 只是「這種攻擊的方式失敗」
- 只要沒有跌破攻擊假設位置 / 頸線突破價，攻擊還在

## 「回測不破」是無稽之談

> 「市場上有一種回測不破的說法，其實**完全是無稽之談**，那是用來安慰不想追高的人使用的藉口，實際上最強的就是根本一路上去不回頭。」

> 「有的人反而因為缺口被回補以為是絕佳的機會，但那就變成是賭會不會再拉上去而已。」

## 真正的攻擊不從低檔起算

> 「認識了跳空攻擊，就會明白真正的攻擊並不是從低檔起來漲一段的那種……而是**創新高價之後的股價拉抬**，才是真正的攻擊走勢。」

## 跳空攻擊的判斷層次

範例：新唐(4919) 110-03-31
- 09:15 盤中（一般散戶不會去追）
- 一開盤跳空，雖大盤偏弱稍震但**不回補缺口**
- 收盤鎖漲停 → 確認跳空攻擊
- 一般人感覺已漲 10%，隔天不敢買 → **抓不到整段**

範例：圓展(3669) 109-07-15
- 突破當日不是長紅（不明顯的突破）
- 隔天開高又往下回檔
- 投資人因兩週前經驗有「先賣有低再買回」心理
- **但仔細看：當日缺口完全沒有回補**
- 即使回補也應該用頸線作為突破後停損
- 還沒到停損為什麼急忙出場？= 低買高賣心理作祟

## 心理鐵則

> 「**買點的重要性遠遠大於賣點**，買錯了位置後面的應對往往會跟著錯誤，買對了位置才有辦法應對強勢股的表現。」

> 「對攻擊型態認識的越深，就會越理解**買黑賣紅、強勢股等拉回買進這些散戶用語，完全與攻擊型態的理論相違背**。」

## 對系統的啟示

### 1. 跳空攻擊精確 entry signal

```python
def is_gap_attack(df):
    """
    跳空攻擊 = 突破紅K隔日 + 跳空 + 收盤不回補
    
    Step 1: 前一日是突破紅K (close > prior_high_60, red)
    Step 2: 今日開盤跳空 (open > prev_high)
    Step 3: 今日收盤未回補缺口 (low > prev_high)
    """
    g = df.groupby("ticker")
    prev_breakout_red = (g["close"].shift(1) > g["prior_high_60"].shift(1)) \
                       & (g["close"].shift(1) > g["open"].shift(1))
    today_gap_up = df["open"] > df["prev_high"]
    today_no_fill = df["low"] > df["prev_high"]
    return (prev_breakout_red & today_gap_up & today_no_fill).fillna(False)
```

→ 這應該作為 **scanner 加分項**或獨立 entry，不只是 detect。

### 2. 跳空攻擊失敗 = 早期 exit

```python
def gap_attack_failed_intraday(df):
    """
    跳空攻擊失敗 = 開盤跳空但收盤回補缺口
    
    僅供事後判斷（盤中無法 vectorize；但 close < prev_high 即為失敗）
    """
    today_gap_up_open = df["open"] > df["prev_high"]
    today_close_fills = df["close"] <= df["prev_high"]
    return today_gap_up_open & today_close_fills
```

→ 失敗時不必立刻 exit（如果還未跌破頸線突破價），但 ranking 應扣分。

### 3. 「回測不破」明確禁止

加入禁止 pattern 清單：
- ❌ 突破後等待「拉回不破突破點」再進場
- ❌ 任何「回測支撐再買」邏輯

### 4. 強化 EXIT_PRIORITY

跳空攻擊失敗 vs 頸線跌破 vs 其他出場條件的排序：
- 跳空攻擊失敗 ≠ 即出場（只是警示）
- 必須結合 `breakout_price_break`（跌破突破價）才是真正出場

### 5. 心理鐵則寫入文檔

> 「買點的重要性遠遠大於賣點」

→ 系統優化方向：**先優化 entry filter**，再優化 exit timing。
（這驗證了之前的判斷：精確頸線的入場 filter 是最大 ROI 改進）

### 6. 真正的攻擊「不從低檔起算」

→ scanner 排除「最近 N 天有破底」的個股（即使後續反彈）

```python
def recent_breakdown_exclude(df, lookback=20):
    """過去 N 天有破底紀錄 → 不算攻擊"""
    g = df.groupby("ticker")
    recent_min = g["close"].rolling(lookback).min().reset_index(level=0, drop=True)
    rolling_low_60 = g["low"].rolling(60).min().reset_index(level=0, drop=True)
    has_recent_breakdown = recent_min < rolling_low_60.shift(lookback)
    return has_recent_breakdown
```
