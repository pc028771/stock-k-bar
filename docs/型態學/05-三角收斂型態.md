## 核心立場：拼湊出來的型態

> 「三角型態是**拼湊出來的**，是透過上方的壓力線、下方的短期趨勢線組合而成。」

**本質**：實務上偏重「**不要怎樣做**」的功能，沒有「應該怎樣做」的邏輯。

## 三角的三種分類（不必細分）

- 對稱三角
- 下降三角（下緣水平 + 上緣下降）
- 上升三角（上緣水平 + 下緣上升）

→ **拆解後就是兩條線**：
- 上緣 = 壓力判斷
- 下緣 = 短期趨勢線

## 操作邏輯：不宜邏輯

> 「如果股價跌破了三角收斂的下緣，那就**暫時不宜作多**；換言之，如果突破了上緣，那就**不宜放空**的意思。」

→ 三角收斂**不是買賣訊號**，只是「短期不宜」的型態判斷。

## 下降三角：跌破 = 頸線跌破

範例：禾伸堂(3026) 108-04-29
- 高檔無轉折組合 + 無攻擊走勢
- 跌破三角下緣 = 不宜進場做多

範例：嘉晶(3016) 107-08-09
- 六月已先有空頭吞噬（多單已該出場）
- 跌破三角下緣 = 頸線跌破

> 「站在三角收斂下緣的意義來說，如果跌破了**等於幾乎是頸線跌破**，這也是為什麼型態學偏重在頭部型態的原因。」

## 上升三角：突破上緣 = 頸線突破

範例：智邦(2345) 108-08-07
- 「低點越來越高」（主力收貨狀態）
- 短期震盪但不讓人有低點買進
- 上緣 = 壓力線 = **頸線**
- 突破三角 = 突破頸線

→ 證明「底部型態沒有意義」的論點：等到股價突破才能確認，那麼**頸線就足夠了**。

## 攻擊意圖的補強

範例：大成(1210) 109-07-15
- 短期收斂三角 + 突破前高
- 「攻擊假設」已成立
- 三角下緣**輔助體會**「主力不給他人有更低進場的意圖」

→ 收斂三角用來**調整投入資金部位大小**。

## 對系統的啟示

### 1. 不需要實作「三角偵測」

依課程立場：拆解就是兩條線（壓力 + 短期趨勢），這兩個別的東西早已涵蓋：
- 壓力線 = 頸線 / overhead supply
- 短期趨勢線 = 多空波動原理

### 2. 但「低點越墊越高」（上升三角下緣）很有用

可作為 scanner 加分因子：

```python
def rising_lows_count(df, lookback=10):
    """連續低點越墊越高的天數"""
    is_higher_low = df["low"] > df["low"].shift(1)
    return is_higher_low.groupby(df["ticker"]).rolling(lookback).sum()
```

→ 這個 feature 已存在於入門系統（`higher_low_count`）！

### 3. 「跌破短期趨勢線」= 早期警示

如果有 swing-low detector：
- 連續 N 個 swing low 形成上升線（最小二乘擬合 slope > 0）
- close 跌破此擬合線 → 早期警示

但課程立場：**這個只是輔助，主要判斷還是用多空波動原理**。

### 4. 加碼大小調整

> 「收斂三角型態作為很基礎的型態來說，還是有正面的幫助的」用於**調整投入資金部位大小**。

→ 對應 scanner ranking：除了 binary entry signal，還可以給「加碼信心度」分數
- 過去 N 天 higher_low_count 高 = 加碼信心高
- 配合 prior_high_60 在收斂中 = 主力收貨確認

### 5. 鐵則：型態學的「不宜」優先於「應該」

> 「**重點不在於買賣點，而在於判斷出不宜為之的型態**。」

→ 系統設計上「過濾不宜進場」比「找買點」更重要。
