## 攻擊跳空的精確定義

> 「股價突破，隔天之後，開始出現了跳空缺口，而**這個價位在過去的歷史成交紀錄中並沒有過**，也就是股價創新高『之後』出現的跳空，才稱之為攻擊跳空。」

關鍵：
- 跳空缺口的價位**過去都沒有成交過**
- 跳空在「突破之後」（先突破，再跳空）

## 兩種形式

| 名稱 | 定義 |
|---|---|
| **突破跳空** | 突破之後**隔天**出現攻擊跳空（理想型） |
| **跳空突破** | 用跳空來突破新高（一開盤就突破）— **不理想** |

**跳空突破為什麼不理想**：
> 「股價要化解賣壓的時候**不需要讓游離籌碼認為股價很強還可以續抱**，這就是跳空突破不理想之處。」

## 攻擊跳空缺口的精確範圍

> 「這個跳空的區段，不是以這根K線的缺口來算，而是**其中以真正過去都沒有成交過的價位區段視為攻擊跳空的缺口**。」

範例：黃圈是整個跳空缺口，但只有橘圈內的部分（真正未成交價位）才算攻擊缺口。
→ 跌破當下看似缺口沒回補，但**攻擊缺口已被回補**。

## 攻擊跳空後的核心判斷

> **「這個缺口不能回補」**

- 被回補 → 「這次沒有打算要攻擊」
- 沒被回補（即使拖很久）→ 攻擊意義依然存在
- 但**拖越久越不對**，因為沒有資金攻擊意圖是要買來拖著的

## 攻擊缺口下緣 = 停損點

> 「攻擊跳空缺口低點就是短線交易者的停損點，停損明確就表示風險固定。」

短線交易者的明確停損：跌破攻擊缺口下緣 → 出場。

## 突破隔日 vs 突破後一段時間

- 突破隔天就攻擊跳空 → 強勢
- 突破後一段時間才出現攻擊跳空 → 此時進場風險較大（但操作上沒有對錯，只是停損位置不同）

## 心法

> 「短線交易並不是要賣在當天高點，而是要操作攻擊段。」

## 對系統的啟示（極重要 — 直接補完入門 E1 缺口）

1. **`gap_fill` 精確化**（取代目前 `excess_gap >= 2%` 粗代理）：
   ```python
   # 攻擊跳空條件
   has_attack_gap = (
       df["open"] > df["prev_high"]            # 跳空
       & (df["close"] > df["prior_high_60"])   # 創新高
       & (df["overhead_supply_layer"] <= 0)    # 左側無成交（無套牢）
   )
   
   # 攻擊缺口下緣 = 過往歷史最高價（prev_high）
   attack_gap_lower = df["prev_high"].where(has_attack_gap)
   
   # 跌破攻擊缺口下緣 → 出場
   exit_when_below_attack_gap = df["close"] < attack_gap_lower
   ```

2. **「跳空突破」vs「突破跳空」**：scanner 可區分：
   - 突破跳空（先突破後跳空）= 強訊號
   - 跳空突破（一日內完成）= 弱訊號（甚至扣分）

3. **攻擊缺口下緣的精確判斷**：
   - 不是「整根 K 線的最低點」
   - 是「攻擊缺口的下緣」= 跳空前一日的 high（prev_high）

4. **拖久 = 失敗**：可加 timeout（如攻擊跳空後 N 天無新高）

5. **歷史價位查找**：需要 `is_price_unprecedented(price)` —— 此價位是否過去從未成交過。可用 prior_high_60 / prior_high_240 等 rolling max 作代理
