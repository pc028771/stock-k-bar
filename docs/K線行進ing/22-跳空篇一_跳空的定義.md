## 跳空缺口定義

**收盤後**：最低價與前一日最高價的距離（如果不重疊就是缺口）
**盤中**：開盤跳空但交易過程可能回補讓缺口消失

## 缺口的位置 > 缺口的大小

> 「缺口出現的『位置』意義遠比『缺口的大小』還要重要，尤其是向上的跳空缺口出現在創新高時、向下跳空缺口出現在創新低時。」

跳空 vs 十字線：
- 十字線 = 醞釀或中繼
- **跳空 = 再一次力量的起點**

## 跳空力量的不對稱性

- **向上跳空**：當天有「不計代價」的買進作為 → 力量
- **向下跳空**：股價疲弱時遇利空，**沒人承接** ≠ 空方力量發揮

> 「以力量發揮的定義來說並不算。」

## 「突破缺口、逃逸缺口、竭盡缺口」理論的問題

> 「並不是這個說法的理論基礎有問題，而是**沒有使用的效果**。」

問題：
- 整個跌勢只有一個向下跳空時該怎麼判斷？
- 漲勢出現三次以上跳空又該如何應對？
- 行進判斷時，如果第三個缺口出現了就出場，後來繼續往上跳空怎麼辦？

範例：大學光出現很多次跳空（最後股價漲到 319 元）— 缺口理論無法用。

> 「事實上也沒有任何拉抬資金是去算已經有幾個缺口來決定股價還要不要拉。」

## 跳空行進判斷的四個確認

1. **攻擊缺口 vs 一般缺口**
2. **是否有利多/利空消息**（vs 無預警出現）
3. **位置**（創新高 / 創新低 / 中間區段）
4. **環境背景**（大盤是盤整還是強勢多頭）

> 「跳空與一般的紅黑K行進判斷不同之處，就是得要再加上環境的背景。」

例：大盤盤整時的個股向上跳空 > 大盤強勢多頭時的向上跳空（力量意義更強烈）

## 對系統的啟示

1. **跳空評分需 context-aware**：
   - 純跳空大小 = 弱訊號
   - 跳空 + 創新高 + 大盤盤整 = 強訊號
   - 跳空 + 創新低 + 個股已疲弱 = 不是空方力量（弱訊號）
2. **禁止「缺口理論」計數**：scanner 不應加入「過去 N 天有幾個缺口」的因子
3. **市場調整跳空**（已有 `market_open_ret`）：用大盤跳空調整個股跳空，與課程「環境背景」邏輯一致
4. **下次系統需新增 feature**：
   - `is_new_high_gap`：向上跳空 + 創新高
   - `is_new_low_gap`：向下跳空 + 創新低（用於警示但不作放空訊號）
