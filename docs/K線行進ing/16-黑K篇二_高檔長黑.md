## 高檔長黑的判斷簡化

高檔長黑只看一個要點：**最高價後來有沒有再突破？**

- 沒突破 → 都是套牢區，介入需謹慎
- 突破 → 變成「再攻擊與否」的判斷

## 高檔的定義（無數字）

> 「**高檔這兩個字沒有辦法用數字或者比例來定義，所以只能回顧過去有沒有拉抬漲勢出現過。**」

關鍵在於：股價經過明顯拉抬之後。

## 不論有沒有包覆紅K

**「黑K吞噬」不是要點**：
- 重點是「在強勢背景下，為什麼輕易回檔成黑K？」
- 包覆紅K（吞噬）：典型訊號
- **未創新高的高檔長黑**：力量的變形，同等警訊
- 「不管是什麼樣的原因，長黑的出現都不是好事，股價變便宜了不代表就可以拉回逢低承接。」

## 「下一個買點」

出場後 → 等到再突破出現攻擊才考慮再進場。

> 「越過高點還不夠，得等到有攻擊出現才能考慮要不要再進場。」

## 高檔長黑 = 中期壓力區

> 「最佳的判斷模式就是股價尚未再突破之前，就是視為這個位置是壓力區域。」

範例欣興(3037)：三次來到同一價位都過不去，第一次就是高檔長黑的日子 → **明顯的壓力訊號**。

## 認識黑K > 認識紅K

> 「**認識黑K遠比認識紅K來得重要太多**，因為買點是最重要的，買了錯誤的位置就等於失去了以後的其他機會。」

「一眼辨識不宜買進的股價K線」 = 高檔長黑的核心功能。

## 對系統的啟示

1. **`high_long_black` 出場條件**（補入門系統缺）：
   - 條件：股價處於「拉抬過後高檔」（如 prior_high_60 已經創過幾次新高）
   - 黑K + body_pct >= 某閾值（如 0.04）
   - **不必有吞噬條件**——只要高檔長黑就是訊號
2. **高檔長黑 = 中期壓力標記**：可在 `features.py` 加 column `high_long_black_price`，記錄歷史高檔長黑的價位作為壓力區
3. **scanner 入場過濾**：候選股近期曾出現高檔長黑 → 扣分或排除（除非已經突破該高檔長黑的價位）
4. **「下一個買點」邏輯**：突破歷史高檔長黑價位 + 攻擊出現 → 才算重新進入攻擊狀態
