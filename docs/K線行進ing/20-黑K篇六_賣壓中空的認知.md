## 賣壓中空定義（精確）

**從形態上的頸線開始，跌破之後出現急跌段（不一定要跌停，連續黑K線也算）**

> 「假如不是連續的下跌，而是上上下下，那就表示成為層層套牢區，這樣就沒有賣壓中空的意義了。」

## 賣壓中空 vs 層層套牢

| 結構 | 條件 | 意義 |
|---|---|---|
| **賣壓中空** | 跌破頸線後**連續急跌**（黑K連續或跌停連續） | 中段無嚴重賣壓 |
| **層層套牢** | 跌破後上上下下盤跌 | 每個價位都套牢 |

範例：鴻海(2317) 107年8月 → 層層套牢的典型

## 賣壓中空為什麼形成

兩種原因：
1. 大環境與個股同時呈現空頭走勢
2. 個股有著自己的重大利空

→ 後續利空解除，但**當初高檔套牢籌碼還是沒人願意幫忙解套**。

## 賣壓中空區段內的攻擊

股價突破低檔整理型態 → 出現攻擊：
- 可以暫時不考慮賣壓
- 但到了「陳年的套牢區」依然會遇到阻礙
- 在套牢區以下 = 賣壓中空區段

→ 反彈很容易停在套牢區下緣，因為「**拉抬資金沒有意願為過往的人解套**」。

## 應對心態

> 「往往遇到了環境的重大利空，**股價快速的下跌比慢慢緩緩盤跌來得好**，因為快速的下跌容易形成未來賣壓中空的結構。」

→ 急跌反而是好事（與「假性跌破」一致背景）

> 「當你理解的其中的意義，就會回頭去思考過去發生趨勢轉空的時候，**一開始就得要當機立斷出場**，否則不管你抱得再久，股價都很難解套。」

## 對系統的啟示

1. **賣壓中空精確判定**：
   - 頸線之下
   - 跌破當時是「連續黑K」或「連續跌停」（**不是上下盤跌**）
   - features 可加 column `is_supply_vacuum`，判斷標準：
     - 前 N 天中 X 天以上連續黑K
     - 期間沒有反彈站回前高
2. **層層套牢的反面**：目前 `overhead_supply_layer` 計算的是「上方 peak count」，正確的賣壓中空需要看「下跌過程中是否連續」
3. **個股反彈遇阻位置 = 歷史套牢區下緣**：可作為 scanner 中「上檔空間」評估
4. **與目前 `supply_vacuum_zone` 對齊**：現有 `supply_vacuum_zone` 用「上方總量 vs 近價帶量比例」是 OHLCV proxy，方向正確但定義需以「下跌連續性」為主
5. **急跌 = 好事**：與 `panic_drop` (5 日跌 7% 以上) 邏輯相符，但 panic_drop 是觸發「假性跌破」邏輯，這裡是「未來賣壓中空」結構基礎
