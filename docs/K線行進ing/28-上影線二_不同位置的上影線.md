# 【行進判斷】上影線(二) 不同位置的上影線

**Order:** 28 / 39 — **HIGH PRIORITY for shadow_position scoring**

## 三種位置的上影線判斷

1. 剛創新高的上影線
2. 已經在攻擊階段的上影線
3. 遇到上方有壓力的上影線

## 剛創新高上影線的兩種子類型

### 子類型 A：上影線低點沒跌破頸線
- 買突破的人**不需要停損**
- 上影線代表攻擊力量，下次再拉就有

### 子類型 B：突破新高後當天跌破頸線，依然留下上影線
- 買突破的人**得先出場**
- 但若隔日**沒有跌破上影線那一根的低點** → 可進場（買在攻擊的「非」常態邏輯）

範例：康舒(6282)、精材(3374) — 上影線低點跌破，攻擊失敗。
範例：優群(3217) — 突破又跌破前高，但上影線低點未破，**這檔還是有漲勢**。

## 核心規則

> 「**當股價低於剛創新高的上影線低點就沒有攻擊的意義**，那麼也就不必持有。」

## 突破 vs 攻擊（重要區分）

> 「**突破應該要以『型態的突破』才算是突破，單純只是突破前高，就要用攻擊的角度來判斷。**」

→ 突破與攻擊的判斷點不同：
- 「型態突破」 = 過去有 N 個月整理期（如 3 個月）後突破
- 「單純突破前高」 = 只是價格高於 prior_high_60，需用攻擊力量判斷

## 已經在攻擊階段的上影線

- 目的：研判**何時出場**（停利）
- **不應該還在這時用停損**——表示根本就亂選股票
- 風險：介入「攻擊過後又創新高的上影線」**比剛突破的上影線略大一點**

範例：瀚荃(8103) — 季線已上揚一段時間，連續兩次上影線隔天破低 → 沒攻擊意圖。

## 遇到賣壓區段的上影線

- 股價處於**非多頭、或近期有套牢賣壓**階段
- 上影線**不代表攻擊**
- 與遇壓向下跳空相比：跳空更慘（賣得更低），上影線是真的遇壓
- 結論：「**本來就不應該去買沒有突破或攻擊的股票**」

## 對系統的啟示（補充 `shadow_position` STUB）

```python
def score(df):
    s = pd.Series(0.0, index=df.index)
    
    # 剛創新高的上影線 = 攻擊
    is_new_high = df["high"] > df["prior_high_60"]
    has_upper = df["upper_shadow_ratio"] >= 1.0
    is_first_new_high = is_new_high & ~(df["prior_high_60"].shift(1) > df["close"].shift(1).rolling(20).max())
    # ↑ 簡化代理「剛」：今天創新高 + 過去 N 天沒創過
    
    s += np.where(is_first_new_high & has_upper, 10, 0)
    
    # 攻擊已久後的上影線 = 警示
    is_late_new_high = is_new_high & ~is_first_new_high
    s += np.where(is_late_new_high & has_upper, 5, 0)  # 較低正分
    
    # 套牢區的上影線 = 壓力
    is_at_overhead = df["overhead_supply_layer"] >= 1
    s -= np.where(is_at_overhead & has_upper & ~is_new_high, 10, 0)
    
    return s
```

1. **「上影線低點跌破」是出場訊號**：
   - 條件：前 N 天有「剛創新高的上影線」
   - 觸發：今日 close < 該上影線那根的 low
2. **區分「剛突破」vs「已攻擊一段」**：用 prior_high_60 vs prior_high_240 對比
3. **「型態突破」vs「單純破前高」**：features 加 column `is_pattern_breakout`（過去 N 個月有整理區間，今日突破整理上緣）
