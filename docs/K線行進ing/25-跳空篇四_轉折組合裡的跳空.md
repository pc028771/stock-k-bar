## 轉折組合裡的跳空 = 一開盤就確認轉折

> 「這個跳空出現的時候，**一開盤就已經知道已經符合轉折的要件**，所以不需要等到收盤再來確認、隔天才出場。」

## 三種空方轉折組合（含跳空缺口）

### 跳空反轉

> **前一天仍是長紅K或創新高格局，但隔日後先出現走低收黑，再隔日的走勢是跳空向下的缺口。**

結構（3 根 K）：
- K1：長紅K 或 創新高
- K2：黑K（從 K1 走低）
- K3：向下跳空

### 雙鴉躍空

> **在高檔壓力區附近跳空開高卻收黑，隔日又出現小黑K（形成雙鴉），隔日再以開盤向下跳空作為確認。**

結構（3 根 K）：
- K1：高檔壓力區附近，跳空開高卻收黑
- K2：小黑K（形成「雙鴉」）
- K3：開盤向下跳空（確認）

關鍵差異於跳空反轉：**不是經過拉抬**，而是**漲勢過後遇到過往的套牢壓力區段**。

### 島狀反轉

> **先跳空向上形成左方缺口，之後再以跳空下跌出現右方缺口，中間形成孤島或不只一根K線的島狀型態。**

關鍵：右邊的缺口向下跳，**一併把左邊的向上跳空力量化解**了 → 形成島狀反轉。

## 個股 vs 大盤的差別

- **個股套牢明確**：必須有買盤願意化解
- **大盤可繞過**：跌了大立光、鴻海，可以拉台積電越過大盤壓力區（資金不用直接面對套牢沉重的個股）
- **大盤的「獲利了結賣壓」可以感受**（不只是套牢）：萬點之上向下跳空 = 獲利了結賣壓的呈現

## 對系統的啟示

1. **`gap_reversal` 精確結構**（取代目前 stub）：
   - K1: 紅K 或 創新高
   - K2: 黑K（close < K1 close）
   - K3: 向下跳空（open < K2 low）
2. **`two_crows` 精確結構**（取代目前 stub）：
   - K1: 高檔壓力區附近 + 跳空開高 + 黑K（這個高檔壓力區可用 `overhead_supply_layer >= 1` 判定）
   - K2: 小黑K
   - K3: 開盤向下跳空（open < K2 low）
3. **`island_reversal` 新增**（入門未提，多空轉折也應補）：
   - 過去 N 天有向上跳空（gap_up_high）
   - 今日向下跳空，且**回補了左邊的向上跳空力量**
4. **轉折組合確認 = 開盤即觸發**：不必等到收盤確認，simulator 應支援「open-time exit」（目前只有 close-time）
5. **dark_double_star（暗夜雙星）也是轉折組合**：與這三個相同，**開盤即確認**——目前系統用收盤確認可能晚一日
