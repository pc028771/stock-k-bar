## 上影線的真正定義

> 「上影線代表著盤中『**有出現過拉抬的力量**』。如果是在創新高的位置有上影線，那就是盤中有出現過攻擊的力量。」

→ 上影線**不代表壓力**（顛覆大眾觀念）

## 上影線四種類型（依位置）

1. **一般上影線**
2. **遇壓上影線**（過往套牢區附近）
3. **剛創新高上影線** ← **攻擊力量**
4. **攻擊過後的上影線**

> 「除了在股價攻擊階段出現的上影線之外，**其他位置出現的上影線都只需要視為一般的 K 線就好**。」

## 剛創新高上影線 vs 攻擊已久後的上影線

| | 剛創新高上影線 | 攻擊已久後上影線 |
|---|---|---|
| 意義 | 攻擊資金才剛剛出力就被套 | 沒有「攻擊資金被套牢」的意義 |
| 處理 | 拉抬資金會想辦法自己再拉 | 攻擊成本評估角度不同 |

## 上影線 vs 下影線（核心對比）

> 「上影線的拉抬力量**在收盤的時候被套了**；
> 下影線的拉抬力量**收盤的時候賺錢了**。」

| 影線 | 收盤狀態 | 後續動作可預期性 |
|---|---|---|
| 上影線 | 拉抬力量被套 | **被套了一定會想辦法自己再拉**（可預期） |
| 下影線 | 拉抬力量賺錢 | 無從判斷打算賣出還是繼續拉（**不可預期**） |

→ 這是「下影線不代表支撐」的根本理由。

## 圖示禁忌

> 「『上影線後面看跌、下影線後面看漲』，才四個字就已經看得出來畫這張圖的人觀念也是錯誤的。」

## 對系統的啟示（直接補完 `shadow_position.py` STUB）

```python
def score(df):
    # 上影線僅在「剛創新高」位置才有正面意義
    is_at_new_high = df["high"] > df["prior_high_60"]
    has_upper_shadow = df["upper_shadow_ratio"] >= 1.0
    
    is_attack_attempt = is_at_new_high & has_upper_shadow & df["is_red"]
    # 攻擊嘗試 → 正面分數
    s = pd.Series(0.0, index=df.index)
    s += np.where(is_attack_attempt, 10, 0)
    
    # 套牢區上影線 → 負面（壓力）
    is_at_overhead = df["overhead_supply_layer"] >= 1
    is_pressure_shadow = is_at_overhead & has_upper_shadow & ~is_at_new_high
    s -= np.where(is_pressure_shadow, 5, 0)
    
    # 下影線：忽略（不加分也不扣分）— 課程明確不給支撐意義
    return s
```

1. **上影線在新高 = 攻擊**（正分）
2. **上影線在套牢區 = 壓力**（負分）
3. **下影線：完全忽略**（不論位置）
4. 「剛」創新高的判定可用 prior_high_N 中 N 較大（如 prior_high_240）以區分「剛」與「已久」
