## 2. 攻擊成本顯現日定義（綜合解讀）

### 必要條件（老師明示，三條缺一不可）

1. **股價突破前高**（即「創新高」，含波動狀態下再創新高）
2. **當日鎖住漲停板**
3. **漲停價就是當日最大量**（成交量越大、成本意義越高）

### 加強條件（老師明示）

- **第一次突破前高** → 攻擊成本意義「最為關鍵」
- **利空當日突破前高漲停鎖住** → 表示強烈企圖，**不用再看攻擊成本**（升級為更強的訊號）

### 排除條件（老師明示）

- **漲太多、已經不是第一次突破前高** → 不在此限（攻擊成本判斷可放寬）
- **股價已經漲到沒有散戶願意佔便宜的點**（脫離基本面飆漲）→ 攻擊成本判斷意義很小（類型三）
- **就算跌破前一天漲停價、散戶還是不會去買**（少數狀態）→ 不需採用攻擊成本，回到攻擊假設

## 3. 連續漲停的分類（核心問題）

### 老師對「攻擊成本顯現日的隔天又漲停」的明示

老師在篇 20 用 **3693 營邦 04-11 / 04-12** 親自示範：
- 04-11 = 攻擊成本顯現日（突破前高 + 鎖漲停 + 最大量在漲停價）
- 04-12 = 「跳空攻擊」= **攻擊成本浮現之後、明日K線『繼續攻擊』的最佳解答**

→ 老師原話定性 04-12 為「**繼續攻擊**」、「**跳空攻擊**」，並說「**至此已經不用再判斷會不會轉變**，而是開始設定移動停利」。

### 答案：B（攻擊企圖確認，不是新的 attack_cost_displayed）

- 04-12 在課程語境是「攻擊企圖的呈現」（跳空攻擊 / 推升攻擊之一），屬於 **04-11 攻擊成本顯現的後續確認動作（branch action）**
- **不是**一個新的 `attack_cost_displayed`
- 理由（從老師原話推導）：
  1. 攻擊成本顯現的判斷重點是「突破前高 + 漲停 + 最大量」，老師強調「**這一點在股價第一次突破前高時，最為關鍵**」。連續漲停的第二天已不是「第一次突破前高」，攻擊成本意義被老師明文排除「漲太多已經不是第一次突破前高的，就不在此限」。
  2. 老師在篇 20 把 04-12 定性為「跳空攻擊 = 攻擊成本浮現之後的『繼續攻擊』最佳解答」，明確歸類為**後續攻擊企圖**、不是新的攻擊成本顯現。
  3. 老師在篇 20 後段說「至此**已經不用再判斷會不會轉變**，而是開始設定移動停利」，表示攻擊狀態已確認、進入「攻擊結束之判別」階段（呼應篇 38），不會再重複觸發 setup-stage 訊號。

### 答案 A、C 為何不採？

- A（連續多次 attack_cost_displayed）：與老師「第一次突破前高最為關鍵 / 漲太多已經不是第一次突破前高的不在此限」原話衝突。
- C（視位置而定）：老師原話沒有給「位置 / 量能 / 是否第一次突破」三維分流的明文規則來把第二日重新歸為攻擊成本顯現。雖然篇 12 用「亞力 1514 連續三根漲停板」描述事實，但**沒有**把第二、三根漲停定義為新的攻擊成本顯現日。

## 5. 建議（給工程層參考，本 note 不執行）

### 偽碼（依老師原話可直接寫的部分）

```
def is_attack_cost_displayed(bar, history):
    # 必要三條（老師明示）
    if not (broke_prior_high(bar, history)
            and locked_limit_up(bar)
            and max_volume_at_limit_price(bar)):
        return False

    # 老師明示排除：已不是第一次突破前高 → 排除
    if not is_first_breakout_of_prior_high(bar, history):
        return False  # 老師原話：「漲太多已經不是第一次突破前高的，就不在此限」

    return True
```

### 需要釐清的 STUB（老師沒給數字 / 邊界）

1. **STUB-NEED-USER #1**：「第一次突破前高」的 lookback 是多久？
   - 老師只說「漲太多已經不是第一次突破前高的，就不在此限」，沒給天數 / 漲幅閾值。
   - 候選實作：（a）自上一次「攻擊結束」之後算第一次；（b）N 日內未曾突破前高；（c）距上次突破前高的漲幅 < X%。
   - 需 user 決策。

2. **STUB-NEED-USER #2**：「漲停價就是當日最大量」如何量化？
   - 老師說「成交量越大、成本意義越高」但沒給門檻。
   - 候選：（a）漲停價的分價成交量 = 當日最大；（b）漲停價分價量占比 ≥ X%。
   - 需 user 決策（建議搭配 Fubon 分價量表）。

3. **STUB-NEED-USER #3**：04-12 應觸發什麼新 pattern？
   - 老師明示 04-12 是「跳空攻擊 = 繼續攻擊的最佳解答」，但 detector 是否需要一個獨立 pattern `attack_intent_confirmed_gap_up` 來標記？還是只用 state machine（attack_cost_displayed → in_attack_phase）抑制？
   - 需 user 決策。

4. **STUB-NEED-USER #4**：利空當日突破前高漲停鎖住 → 老師說「不用再看攻擊成本」、是更強訊號。
   - 是否需要一個 `attack_cost_displayed_under_bad_news` 高優先級分支？老師沒給具體運作差異。
   - 需 user 決策。

### 不需 STUB、可直接實作的規則

- **狀態機抑制**：一旦觸發 `attack_cost_displayed`，進入 in_attack_phase，後續連續漲停 / 跳空攻擊 / 推升攻擊歸為「攻擊企圖確認」branch，不再亮 setup-stage 訊號；直到「跌破攻擊成本」或「攻擊結束（趨勢線跌破 / 波動前低跌破）」才退出狀態。
- **第一次突破前高 gating**：在 `attack_cost_displayed` 條件加入 `is_first_breakout_of_prior_high`（搭配 STUB #1 的定義）。
