## 關鍵 insights

### 1. `main_force_20d ≥ 100 萬` 是「砍掉大虧」的核心 filter
- mf20 ≥ 1M 時，**所有 dev_240 切點的 worst case 從 -11.63% 收斂到 -0.45 ~ +0.36%**
- 課程 §九.1 hard-coded 要求 mf 全 > 0，但**強度沒明示**；實證顯示「強度」比「正負號」更關鍵
- 意義：主力 20 日累積買超弱（< 100 萬）= 短期隔日沖刻意製造的爆量，非真主力進場

### 2. `dev_ma240_pct ≤ 10%` 是「剛起漲」的真正定義
- 課程 §九 示範 30%，但 30% 切點 win 只有 56.5%，worst -11.63%
- 實證 ≤ 10% 才能避免「追高」假突破
- 對應 ch3-2 @00:23 課程原文「**剛起漲離年線不遠**」的精神（30% 已經漲很多了）

### 3. 兩個 filter 對 worst case 的影響是疊加的
- 單獨 dev_240 ≤ 10%：worst -11.63%
- 單獨 mf20 ≥ 1M：worst -3.74%（dev=20 時）
- **兩者並用**：worst +0.36%（dev=10 + mf20=1M）

## 推薦組合（按 user 目標：低樣本 + 高勝率 + 低停損）

### 🥇 最嚴格版 (`configs/strict_high_quality.json`)
```json
{
  "lixia_dev_240_max_pct": 10.0,
  "lixia_mf20_min": 1000000
}
```
- 3 筆 / 17 個月（≈ 0.18 筆/月）
- **100% win**, mean +10.55%, **worst +0.36%（無虧損）**
- 適合：保守取向、嚴選最高品質

### 🥈 平衡版 (`configs/balanced_high_quality.json`)
```json
{
  "lixia_dev_240_max_pct": 10.0,
  "lixia_mf20_min": 500000
}
```
- 5 筆 / 17 個月（≈ 0.29 筆/月）
- 80% win, mean +8.21%, **worst 0.00%（最差打平）**
- 適合：要稍多訊號但仍要零虧損保證

### 🥉 寬鬆版 (`configs/loose_quality.json`)
```json
{
  "lixia_dev_240_max_pct": 15.0,
  "lixia_mf20_min": 1000000
}
```
- 8 筆 / 17 個月（≈ 0.47 筆/月）
- 75% win, mean +5.60%, worst -0.45%
- 適合：要更多進場機會，可接受極小虧損
