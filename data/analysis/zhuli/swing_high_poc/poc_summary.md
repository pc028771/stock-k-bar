## 視覺判讀優劣分析

### find_peaks (distance=5, prominence=2%)

**優點：**
- 結合了高度（prominence）門檻，過濾掉小波動的假高點
- 強制間距 5 根，避免相鄰重複點
- 在 60 根窗口輸出 5-8 個點，密度適中，接近「肉眼挑兩個明顯高點」的感覺

**缺點：**
- prominence 是全局均值計算，對股價突然大漲/大跌的個股可能過緊或過鬆
- 聯發科 90 根窗口出現 8 個，仍偏多（主力大課程通常只找 2 個）

**適合程度：** ★★★★☆（最接近主力大肉眼風格）

### rolling max w=5

**優點：**
- 邏輯簡單直觀：高點是「左右各 2 根都不比它高」
- 不需要調參數（除了 window）

**缺點：**
- 太敏感，容易找到相鄰的小高點（聯發科 14 個）
- 沒有振幅門檻，平台整理期間的微小高點都會被選中
- 6237 找出 10/17 這個次要高點（不是明顯頂部）

**適合程度：** ★★☆☆☆（雜訊過多，需後處理）

### rolling max w=10

**優點：**
- 窗口拉大，只保留「左右各 4 根都不比它高」的點
- 輸出數量最少（3-6 個），接近人工判斷的稀疏程度

**缺點：**
- 對趨勢股（持續上漲）容易錯過早期高點
- window=10 邊界效應：窗口末端的資料點因 min_periods 可能不被選到
- 在 3548 長興僅找到 3 個，太少、12/23 可能是誤判（原始高點在 12/20）

**適合程度：** ★★★☆☆（數量合理，但準確率待驗證）

## 建議的下一步

### 1. 推薦演算法：find_peaks 作為基底，搭配後處理

`find_peaks (d=5, prom=2%)` 輸出最接近主力大「肉眼挑明顯高點」的結果：
- 間距約束 + 振幅門檻雙重過濾，噪點少
- 輸出 4-8 個候選點，人工再挑 2 個容易操作
- 建議加一個「最多保留 N 個最高 prominence 的點」後處理（N=3~5）

```python
# 後處理：只保留 prominence 最大的 N 個
peaks, props = find_peaks(highs, distance=5, prominence=threshold)
if len(peaks) > 5:
    top_idx = np.argsort(props["prominences"])[-5:]
    peaks = peaks[top_idx]
```

### 2. Ensemble 思路

可以做 `find_peaks ∩ rolling_w10` 的交集：
- 兩種方法都認可的點，信心更高
- 觀察三個案例：大部分 rolling_w10 的點都出現在 find_peaks 的結果裡
- 交集通常剩 3-4 個，正好適合讓人工或 vision 挑 2 個

### 3. 下一步驗證

1. 使用 `--vision` 旗標（需要 ANTHROPIC_API_KEY）對 PNG 做 vision 判讀，驗證演算法選點是否接近人工判斷
2. 找真實的下降趨勢 + 反轉 K 個股（需有明確「反轉紅K站上下降趨勢線」的歷史案例）來測試演算法能否正確定位那兩個高點
3. 考慮加入「只在趨勢向下時才找高點」的前置過濾（用 MA20 斜率判斷），避免橫盤或上漲期間選到不相關的高點

### 4. 不建議做 rolling_w5 的 ensemble

rolling_w5 雜訊太多，加進去反而拉低精確度。若要用 rolling，直接用 w=10 即可。
